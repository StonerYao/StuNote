Vue

Vue 安装

  • 官网下载或 CDN，通过 <script> 引入。
    Vue 会被注册为一个全局变量。
  • 通过 NPM 安装。
    npm install vue
  • 通过 Vue CLI 安装。

Vue 版本

  • 开发版本 vue.js
    包含完整的警告和调试模式。
  • 生产版本 vue.min.js
    没有警告并使用了压缩。

Vue 模板语法

Vue 使用了基于 HTML 的模板语法。在底层的实现上，Vue 将模板编译成虚拟 DOM 渲染函数。也可直接写渲染 (render) 函数，使用可选的 JSX 语法。

插值

插值使用 Mustache 语法 {{ }} 实现数据绑定。其中的值可以是以下列表中的任意一项：

  • 文本
    {{ msg }}
    Mustache 标签将会被替代为对应数据对象上 msg property 的值，如果没有对应的 msg property 就会报错。无论何时，绑定的数据对象上 msg property 发生了改变，插值处的内容都会更新。
  • JavaScript 数据值
    {{ 1 }} | {{ 'ok' }} | {{ true }} | {{ undefined }} | {{ null }} | {{ ['a', 'b', 'c'] }} | {{ {height: '140', weight: '100'} }}
  • JavaScript 表达式
    {{ number + 1 }} | {{ ok ? 'YES' : 'NO' }} | {{ message.split('').reverse().join('') }}
    JavaScript 表达式会在所属 Vue 实例的数据作用域下作为 JavaScript 被解析。有个限制就是，每个绑定都只能包含单个表达式。

Vue 实例

引入 vue.js 后，它提供了一个构造函数 Vue，通过用 new Vue() 创建一个 Vue 实例，同时可以传入一个选项对象来控制行为，最后再用变量接住这个实例，以此搭建 Vue 应用。另外，所有的 Vue 组件都是 Vue 实例，并且接受相同的选项对象 (一些根实例特有的选项除外)。

Vue 为了实现响应式、监控数据变化、执行某个监听函数，在创建 Vue 实例时，会将 data 中的成员代理给 Vue 实例。同时为了防止名称冲突，Vue 会把自己内部的属性成员名称前加上 $ 或 _。

实例配置

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Vue</title>
  <!-- 引入vue.js提供一个构造函数Vue -->
  <script src="./vue.js" type="text/javascript" charset="utf-8"></script>
</head>
<body>
  <!-- 被Vue控制的区域 模板 -->
  <div id="app">
    <!-- 插值表达式 获取Vue中的数据对象 -->
    {{ a }}
    <!-- 插值表达式 获取Vue中事件处理函数的返回值 -->
    {{ plus() }}
    <!-- 插值表达式 获取Vue中计算属性的返回值 -->
    {{ double minus }}
    <!-- 自定义元素 使用组件 -->
    <component-a></component-a>
    <component-b></component-b>
  </div>
  <!-- Vue脚本 -->
  <script type="text/javascript">
    var ComponentA = { /* ... */ }; // 定义组件
    var ComponentB = { /* ... */ };
    const vm = new Vue({ // 创建Vue实例 传递配置对象
      el: '#app', // 控制id为app的元素
      template: `<div id="ceshi">xxx</div>`, // 配置模板
      data: { // 配置要用到的数据对象
        a: 1,
        b: 2,
        c: 3,
        d: 4,
        e: {
          f: {
            g: 5
          }
        },
        h: {
          i: 6,
          j: 7
        },
        msg: 'hellow world'
      },
      methods: { // 配置事件处理函数
        plus: function () {
          return this.h.i++; // 通过this获取Vue实例的上下文
        },
        handleClick () {
          console.log(handleClick);
        }
      },
      computed: { // 配置计算属性
        double: function () { // 函数形式仅读取
          return this.h.j * 2; // 返回的值就是计算属性的值
        },
        minus: { // 对象形式可读取和设置
          get: function () {
            return this.h.j - 1; // this上下文自动地绑定Vue实例
          },
          set: function (v) { // 通过参数v获取设置值
            this.h.j = v - 1;
          }
        }
      },
      watch: { // 配置要侦听的数据
        a: function (val, oldVal) { // 函数形式 通过参数分别获取新值和旧值
          console.log('new: %s, old: %s', val, oldVal);
        },
        b: 'someMethod', // 字符串形式 配置回调函数
        c: { // 对向形式 通过handler属性配置回调函数 通过deep属性开启对象嵌套属性的侦听
          handler: function (val, oldVal) { /* ... */ },
          deep: true
        },
        d: { // 对向形式 通过handler属性配置回调函数 通过immediate属性开启侦听时的立即调用
          handler: 'someMethod',
          immediate: true
        },
        e: [ // 数组形式 配置会被逐一调用的回调函数
          'handle1',
          function handle2 (val, oldVal) { /* ... */ },
          {
            handler: function handle3 (val, oldVal) { /* ... */ },
            /* ... */
          }
        ],
        'e.f': function (val, oldVal) { /* ... */ } // 配置要侦听对象的嵌套数据
      },
      beforeCreate () { // 配置beforeCreate钩子函数
        console.log(this.msg);   // undefined
        console.log(this.handleClick);  // undefined
      },
      created () { // 配置created钩子函数
        console.log(this.msg);  // hello world
        console.log(this.handleClick);  // function () {...}
        console.log(this.$el);  // undefined
        this.timer = setInterval(() => {
          console.log('xxx');
        }, 500);
      },
      beforeMount () { // 配置beforeMount钩子函数
        console.log(this.$el); // 旧模板
      },
      mounted () { // 配置mounted钩子函数
        console.log(this.$el); // 新模板
      },
      beforeUpdate () { // 配置beforeUpdate钩子函数
        console.log(this.msg); // 新值
        console.log(this.$el.outerHTML); // 旧值
      },
      updated () { // 配置updated钩子函数
        console.log(this.msg); // 新值
        console.log(this.$el.outerHTML); // 新值
      },
      beforeDestroy () { // 配置beforeDestroy钩子函数
        clearInterval(this.timer); // 清除定时器
      },
      destroyed () { // 配置destroyed钩子函数
        console.log('destroyed');
      },
      components: { // 配置要使用的局部组件
        'component-a': ComponentA,
        'component-b': ComponentB
      }
    });
  </script>
</body>
</html>

Vue 数据绑定

基础绑定

<!-- 绑定一个属性 -->
<img v-bind:src="imageSrc">

<!-- 缩写 -->
<img :src="imageSrc">

<!-- 使用动态特性名 -->
<button :[key]="value"></button>

<!-- 通过内联字符串拼接绑定 -->
<img :src="'/path/to/images/' + fileName">

<!-- 不带参数通过一个对象绑定多个属性 -->
<!-- 此时绑定不支持内联数组和对象 -->
<div v-bind="{ id: someProp, 'other-attr': otherProp }"></div>

Class 绑定

<!-- 使用对象语法根据表达式值的 truthiness 有条件地绑定 -->
<div :class="{ active: isActive }"></div>

<!-- 使用对象语法绑定多个 class -->
<!-- 属性名中有特殊分隔符需要用引号 -->
<div :class="{ active: isActive, 'text-danger': hasError }"></div>

<!-- 通过 data 中的 class 对象绑定 -->
<div :class="classObject"></div>

<!-- 使用数组语法绑定多个 class -->
<div :class="[classA, classB]"></div>

<!-- 通过三元表达式控制绑定 -->
<div :class="[isActive ? activeClass : '', errorClass]"></div>

<!-- 嵌套对象语法控制绑定 -->
<div :class="[classA, { classB: isB, classC: isC }]">

<!-- 同时使用普通 class 特性 -->
<!-- 普通 class 与绑定 class 都将更新到 class 列表中 -->
<div :class="classA" class="red">

Style 绑定

<!-- 使用对象语法绑定 -->
<!-- CSS 属性名可以用驼峰式或引号加短横线分隔来命名 -->
<div :style="{ fontSize: size + 'px' }"></div>

<!-- 通过 data 中的样式对象绑定 -->
<div :style="styleObject"></div>

<!-- 使用数组语法绑定多个样式对象 -->
<div :style="[styleObjectA, styleObjectB]"></div>

<!-- 可以为属性提供一个向后兼容的多值数组 -->
<div :style="{ display: ['-webkit-box', '-ms-flexbox', 'flex'] }"></div>

<!-- 当使用需要添加浏览器引擎前缀的 CSS 属性时，Vue.js 会自动侦测并添加相应的前缀。 -->

Vue 事件处理

事件处理方法

<!-- 直接编写事件处理代码并在触发 DOM 事件时调用 -->
<!-- 在普通元素上时只能监听原生 DOM 事件 -->
<!-- 在自定义元素组件上时可以监听子组件触发的自定义事件 -->
<button v-on:click="doThis"></button>

<!-- 缩写 -->
<button @click="doThis"></button>

<!-- 使用动态事件名 -->
<button @[event]="doThis"></button>

<!-- 绑定 methods 中的方法函数作为事件处理代码 -->
<!-- 事件处理函数中接收一个原生 DOM 事件对象作为参数 -->
<button @click="doThat"></button>

<!-- 通过内联 JavaScript 语句为事件处理函数传参 -->
<button @click="doThat('hello')"></button>

<!-- 同时传入事件对象以访问原始的 DOM 事件 -->
<!-- 使用内联语句时可以使用 $event 作为事件对象 -->
<button @click="doThat('hello', $event)"></button>

<!-- 使用修饰符阻止默认行为 -->
<button @click.prevent="doThis"></button>

<!-- 使用串联修饰符阻止对元素自身的点击 -->
<!-- 需要注意修饰符顺序 v-on:click.prevent.self 会阻止所有的点击-->
<!-- .passive 和 .prevent 互斥不可一起使用 -->
<button @click.self.prevent="doThis"></button>

<!-- 使用通过全局 config.keyCodes 对象自定义按键修饰符 -->
<!-- Vue.config.keyCodes.f1 = 112 -->
<input @keyup.f1="doThis">

<!-- 只有在按住 ctrl 的情况下释放其它按键才能触发 -->
<input @keyup.ctrl="doThis">

<!-- 没有任何系统修饰符被按下的时候才触发 -->
<button @click.exact="doThis"></button>

<!-- 不带参数通过一个对象绑定多个事件 -->
<!-- 使用此种方法不支持函数传参和修饰符 -->
<button v-on="{ mousedown: doThis, mouseup: doThat }"></button>

在 HTML 中监听事件的优势

  • 扫一眼 HTML 模板便能轻松定位在 JavaScript 代码里对应的方法。
  • 因为无须在 JavaScript 里手动绑定事件，ViewModel 代码可以是非常纯粹的逻辑，和 DOM 完全解耦，更易于测试。
  • 当一个 ViewModel 被销毁时，所有的事件处理器都会自动被删除。无须担心如何清理它们。

Vue 列表渲染

<!-- 值必须使用特定语法为当前遍历的元素提供别名 -->
<!-- 特定语法可以是 alias in expression 也可以是 alias of expression -->
<div v-for="item in items">
<div v-for="item of items">

<!-- 通过可选的第二个参数获取当前遍历元素的索引 -->
<div v-for="(item, index) in items"></div>

<!-- 通过可选的第二个参数获取当前对象项的键 -->
<!-- 通过可选的第三个参数获取当前对象项的索引 -->
<div v-for="(val, name, index) in object"></div>

<!-- 不使用 key 时 Vue 会使用一种最大限度减少动态元素并且尽可能的尝试就地修改/复用相同类型元素的算法 -->
<!-- 使用 key 时它会基于 key 的变化重新排列元素顺序并且会移除 key 不存在的元素 -->
<!-- key 的值必须是不能重复的字符串或数值类型的值 不要使用 index 作为它的值-->
<div v-for="item in items" :key="item.id">

<!-- 被遍历的源数据可以是数字或字符串 -->
<div v-for="item in 10">
<div v-for="item in 'aaaa'">

Vue 表单控件绑定

<!-- input[text] 元素 绑定 value 属性 监听 input 事件 -->
<!-- v-model 会忽略所有表单元素的 value、checked、selected attribute 的初始值而总是将 Vue 实例的数据作为数据来源 -->
<input v-model="message">

<!-- textarea 元素 绑定 value 属性 监听 input 事件 -->
<!-- 在文本区域插值 (<textarea>{{text}}</textarea>) 并不会生效 应用 v-model 来代替 -->
<textarea v-model="message"></textarea>

<!-- input[checkbox] 元素 绑定 checked/value 属性 监听 change 事件 -->
<input type="checkbox" v-model="checked">
<input type="checkbox" value="Jack" v-model="checkedNames">

<!-- input[radio] 元素 绑定 checked/value 属性 监听 change 事件 -->
<input type="radio" v-model="picked">
<input type="radio" value="One" v-model="picked">

<!-- select 元素 绑定子元素 option 的 value 属性 监听 change 事件 -->
<!-- 如果 v-model 表达式的初始值未能匹配任何选项 <select> 元素将被渲染为“未选中”状态 -->
<select v-model="selected"></select>
<select v-model="selected" multiple></select>

<!-- 即使在 type="number" 时 HTML 输入元素的值也总会返回字符串 -->
<!-- 使用 .number 修饰符如果输入的值无法被 parseFloat() 解析为数值类型 则会返回原始的值 -->
<input v-model.number="age" type="number">

Vue 混入

用于分发 Vue 组件中的可复用功能。一个混入对象可以包含任意组件选项。当组件使用混入对象时，所有混入对象的选项将被“混合”进入该组件本身的选项。

var mixin = {
  created () {
    this.hello();
  },
  methods: {
    hello () {
      console.log('hello，我是混入中的函数');
    },
  }
}

Vue.component('my-cmp', {
  mixins: [mixin],
  template: `
    <div>xx</div>
  `
})

// => "hello，我是混入中的函数"

选项合并

当组件和混入对象含有同名选项时，这些选项将使用和 Vue.extend() 一样的选项合并逻辑进行“合并”。

  • 数据对象在内部会进行递归合并，并在发生冲突时以组件数据优先。

    var mixin = {
  data () {
    return {
      msg: 'hello',
    }
  }
}
new Vue({
  mixins: [mixin],
  data: {
    msg: 'goodbye',
  },
  created: function () {
    console.log(this.msg) // => "goodbye"
})

  • 钩子函数将合并为一个数组按照传入顺序依次调用，混入对象的钩子将在组件自身钩子之前调用。同名钩子函数都将被调用。

    var mixin = {
  created () {
    console.log('混入对象钩子')
  }
}

new Vue({
  el: '#app',
  mixins: [mixin],
  created () {
    console.log('组件钩子')
  }
})
// => "混入对象钩子"
// => "组件钩子"

  • 对于 methods、components 和 directives 值为对象的选项，将被合并为同一个对象。两个对象键名冲突时，取组件对象的键值对。

    var mixin = {
  methods: {
    foo: function () {
      console.log('foo')
    },
    conflicting: function () {
      console.log('from mixin')
    }
  }
}

var vm = new Vue({
  mixins: [mixin],
  methods: {
    bar: function () {
      console.log('bar')
    },
    conflicting: function () {
      console.log('from self')
    }
  }
})

vm.foo() // => "foo"
vm.bar() // => "bar"
vm.conflicting() // => "from self"

全局混入

使用全局注册的混入，将影响每一个之后创建的 Vue 实例。使用恰当时，这可以用来为自定义选项注入处理逻辑。

Vue.mixin({
  created: function () {
    var myOption = this.$options.myOption
    if (myOption) {
      console.log(myOption)
    }
  }
})

new Vue({
  myOption: 'hello!'
})
// => "hello!"

Vue 自定义指令

用于对普通 DOM 元素进行底层操作。

  • 全局注册自定义指令

    Vue.directive('focus', {/** 指令的定义 **/})

  • 局部注册自定义指令

    const vm = new Vue({
  el: '#app',
  directives: {
    focus: {/** 指令的定义 **/}
  }
})

  • 使用自定义指令

    <input v-focus>

  • 返回已注册的指令

    var myFocus = Vue.directive('focus')

钩子函数

  • bind：只调用一次，指令第一次绑定到元素时调用。
    在这里可以进行一次性的初始化设置。

  • inserted：被绑定元素插入父节点时调用。
    仅保证父节点存在，但不一定已被插入文档中。

  • update：所在组件的 VNode 更新时调用，但是可能发生在其子 VNode 更新之前。
    指令的值可能发生了改变，也可能没有。可以通过比较更新前后的值来忽略不必要的模板更新。

  • componentUpdated：指令所在组件的 VNode 及其子 VNode 全部更新后调用。

  • unbind：只调用一次，指令与元素解绑时调用。

钩子函数参数

  • el：指令所绑定的元素。
    可以用来直接操作 DOM。
  • binding：一个对象，包含以下 property：
    • name：指令名，不包括 v- 前缀。
    • value：指令的绑定值。
      例如：v-my-directive="1 + 1" 中，绑定值为 2。
    • oldValue：指令绑定的前一个值，仅在 update 和 componentUpdated 钩子中可用。无论值是否改变都可用。
    • expression：字符串形式的指令表达式。
      例如 v-my-directive="1 + 1" 中，表达式为 "1 + 1"。
    • arg：传给指令的参数，可选。
      例如 v-my-directive:foo 中，参数为 "foo"。
    • modifiers：一个包含修饰符的对象。
      例如：v-my-directive.foo.bar 中，修饰符对象为 { foo: true, bar: true }。
  • vnode：Vue 编译生成的虚拟节点。
  • oldVnode：上一个虚拟节点，仅在 update 和 componentUpdated 钩子中可用。

除了 el 之外，其它参数都应该是只读的，切勿进行修改。如果需要在钩子之间共享数据，建议通过元素的 dataset 来进行。

动态指令参数

<div id="app">
  <div v-mydirective:[argument]></div>
</div>

Vue.directive('mydirective', {
  bind: function (el, binding, vnode) {
    var test = binding.arg;
    console.log(test); // => "foo"
  }
})

new Vue({
  el: '#app',
  data: {
    argument: 'foo'
  }
})

函数简写

用于在 bind 和 update 时触发相同行为，而不关心其它的钩子。

Vue.directive('my-directive', function () {
  // 这里将会被 `bind` 和 `update` 调用
})

对象字面量

当指令需要多个值，可以传入一个 JavaScript 对象字面量。指令函数能够接受所有合法的 JavaScript 表达式。

<div v-demo="{ color: 'white', text: 'hello!' }"></div>

Vue.directive('demo', function (el, binding) {
  console.log(binding.value.color) // => "white"
  console.log(binding.value.text)  // => "hello!"
})

Vue 过滤器

用于将双花括号插值和 v-bind 表达式中的文本格式化。

  • 全局注册过滤器

    Vue.filter('my-filter', function (value) {
  // 返回处理后的值
})

  • 局部注册过滤器

    const vm = new Vue({
  el: '#app',
  filters: {
    'my-filter': function (value) {
      // 返回处理后的值
    }
  }
})

  • 使用过滤器

    <!-- 在双花括号中 -->
{{ message | capitalize }}

<!-- 在 `v-bind` 中 -->
<div v-bind:id="rawId | formatId"></div>

  • 返回已注册的过滤器

    var myFilter = Vue.filter('my-filter')

当全局过滤器和局部过滤器重名时，会采用局部过滤器。

过滤器参数

过滤器函数总接收表达式的值 (之前的操作链的结果) 作为第一个参数。

{{ message | filterA('arg1', arg2) }}

filterA 被定义为接收三个参数的过滤器函数。其中 message 的值作为第一个参数，普通字符串 'arg1' 作为第二个参数，表达式 arg2 的值作为第三个参数。

串联过滤器

{{ message | filterA | filterB }}

filterA 被定义为接收单个参数的过滤器函数，表达式 message 的值将作为参数传入到函数中。然后继续调用同样被定义为接收单个参数的过滤器函数 filterB，将 filterA 的结果传递到 filterB 中。

Vue 渲染函数

利用渲染函数可以利用 JavaScript 的完全编程的能力更加灵活的创建 HTML。

  • 使用模板

    template: '<h1>{{ blogTitle }}</h1>'

  • 使用渲染函数

    render: function (createElement) {
  return createElement('h1', this.blogTitle)
}

render 的优先级高于 template，template 的优先级高于挂载元素的 outerHTML。另外使用 render 可以不用编译直接渲染。

createElement

createElement 函数用于返回一个虚拟节点。 Vue 通过建立由这些虚拟节点组成的虚拟 DOM 来追踪自己要如何改变真实 DOM。

// @returns {VNode}
createElement(
  // {String | Object | Function}
  // 一个 HTML 标签名、组件选项对象，或者
  // resolve 了上述任何一种的一个 async 函数。必填项。
  'div',

  // {Object}
  // 一个与模板中 attribute 对应的数据对象。可选。
  {
    // 与 `v-bind:class` 的 API 相同，
    // 接受一个字符串、对象或字符串和对象组成的数组
    'class': {
      foo: true,
      bar: false
    },
    // 与 `v-bind:style` 的 API 相同，
    // 接受一个字符串、对象，或对象组成的数组
    style: {
      color: 'red',
      fontSize: '14px'
    },
    // 普通的 HTML attribute
    attrs: {
      id: 'foo'
    },
    // 组件 prop
    props: {
      myProp: 'bar'
    },
    // DOM property
    domProps: {
      innerHTML: 'baz'
    },
    // 事件监听器在 `on` 内，
    // 但不再支持如 `v-on:keyup.enter` 这样的修饰器。
    // 需要在处理函数中手动检查 keyCode。
    // 对于 .passive、.capture 和 .once 事件修饰符，
    // Vue 提供了相应的前缀可以用于 on。
    // 除此之外其它的修饰符，要在事件处理函数中手动实现。
    on: {
      click: this.clickHandler,
      '~keyup': this.doThisOnce,
      keyup: function (event) {
        event.preventDefault();
      }
    },
    // 仅用于组件，用于监听原生事件，而不是组件内部使用
    // `vm.$emit` 触发的事件。
    nativeOn: {
      click: this.nativeClickHandler
    },
    // 自定义指令。注意，你无法对 `binding` 中的 `oldValue`
    // 赋值，因为 Vue 已经自动为你进行了同步。
    directives: [
      {
        name: 'my-custom-directive',
        value: '2',
        expression: '1 + 1',
        arg: 'foo',
        modifiers: {
          bar: true
        }
      }
    ],
    // 作用域插槽的格式为
    // { name: props => VNode | Array<VNode> }
    scopedSlots: {
      default: props => createElement('span', props.text)
    },
    // 如果组件是其它组件的子组件，需为插槽指定名称
    slot: 'name-of-slot',
    // 其它特殊顶层 property
    key: 'myKey',
    ref: 'myRef',
    // 如果你在渲染函数中给多个元素都应用了相同的 ref 名，
    // 那么 `$refs.myRef` 会变成一个数组。
    refInFor: true
  },

  // {String | Array}
  // 子级虚拟节点 (VNodes)，由 `createElement()` 构建而成，
  // 也可以使用字符串来生成“文本虚拟节点”。可选。
  // 需要注意，组件树中的所有虚拟节点必须是唯一的不可重复的。
  [
    '先写一些文字',
    createElement('h1', '一则头条'),
    createElement(MyComponent, {
      props: {
        someProp: 'foobar'
      }
    })
  ]
)

对于 v-if v-for v-model 可以在原生的 JavaScript 中轻松完成的操作，Vue 的渲染函数就没有提供专有的替代方法，需要自己实现相应的逻辑。
对于插槽的使用，可以通过 this.$slots 访问静态插槽的内容，也可以通过 this.$scopedSlots 访问作用域插槽。

<anchored-heading :level="1">Hello world!</anchored-heading>

  render: function (createElement) {
    return createElement(
      'h' + this.level,   // 标签名称
      this.$slots.default // 子节点数组
    )
  },
  props: {
    level: {
      type: Number,
      required: true
    }
  }
})

在 Vue 中使用 JSX 语法创建虚拟 DOM，可以让我们回到更接近于模板的语法上。

  • 基础使用

函数式组件

将无状态 (没有响应式数据)，没有生命周期方法，也没有实例 (没有 this 上下文) 的组件定义为函数式组件可以降低渲染的开销。

在组件内定义 functional 选项为 true。

Vue.component('my-component', {
  functional: true,
  // Props 是可选的
  props: {
    // ...
  },
  // 为了弥补缺少的实例
  // render 函数提供第二个参数 context 作为上下文
  render: function (createElement, context) {
    const { props, slots, scopedSlots } = context;
    const tag = 'h' + props.level;
    return (
      <tag>
        { slots().default }
        { scopedSlots.default({ text: '默认插槽'}) }
      </tag>
    )
  }
})

单文件组件在 template 标签上标记 functional。

<template functional>
  <button
    class="btn btn-primary"
    v-bind="data.attrs"
    v-on="listeners"
  >
    <slot />
  </button>
</template>

context 是一个包括如下字段的对象：

  • props：提供所有 prop 的对象。当省略 props 选项时，所有组件上的 attribute 都会被自动隐式解析为 prop。
  • children：VNode 子节点的数组，包含了所有的非作用域插槽和非具名插槽。
  • slots：一个函数，返回了除作用域插槽外所有插槽的对象。
  • scopedSlots：(2.6.0+) 一个暴露传入的作用域插槽的对象。也以函数形式暴露普通插槽。
  • data：传递给组件的整个数据对象，作为 createElement 的第二个参数传入组件。
  • parent：对父组件的引用。
  • listeners：(2.3.0+) 一个包含了所有父组件为当前组件注册的事件监听器的对象。这是 data.on 的一个别名。
  • injections：(2.3.0+) 如果使用了 inject 选项，则该对象包含了应当被注入的 property。

children 和 slots() 对比：

示例1：

<base-level :level="1" @click="handleClick">

  </template>

Vue 单文件组件

单文件组件是将一个组件全部内容汇合到一个扩展名为 .vue 的文件中。使用单文件组件具有以下优势：

  • 支持 CommonJS 模块，使得组件更加内聚且更可维护，且不用担心全局定义的命名重复。
  • 提供完整语法高亮，也不用关注字符串模板中遇到的换行问题。
  • 获得组件作用域的 CSS，不用担心组件化时样式的缺失。
  • 有构建步骤限制，从而能使用预处理器，如 Pug 和 Babel。
  • 可以使用 Vue CLI 脚手架，也可以使用 webpack 或 Browserify 等构建工具。

在单文件组件中仍然可以把 JavaScript、CSS 分离成独立的文件然后做到热重载和预编译。

<!-- my-component.vue -->
<template>
  <div>This will be pre-compiled</div>
</template>
<script src="./my-component.js"></script>
<style src="./my-component.css"></style>

Vue 响应式原理

追踪变化

当把一个普通的 JavaScript 对象传入 Vue 实例作为 data 选项，Vue 将递归遍历此对象所有的 property，并使用 Object.defineProperty 把这些 property 全部转为 getter/setter，在内部它们让 Vue 能够追踪依赖，在 property 被访问和修改时通知变更，从而使页面重新渲染。

每个组件实例都对应一个 watcher 实例，它会在组件渲染的过程中把“接触”过的数据 property 记录为依赖。之后当依赖项的 setter 触发时，会通知 watcher，从而使它关联的组件重新渲染。

Vue 的响应式系统会在修改数据对象时将视图更新。但由于 JavaScript 的限制，Vue 不能检测数组和对象的变化。

在对象中，对于已经创建的实例，Vue 不允许动态添加根级别的响应式 property，也监听不到对象的新增 property。但可以使用 Vue.set 或 vm.$set 方法向嵌套对象添加 property 并触发更新视图。并且 Vue 不能检测使用 delete 操作符删除对象的某个属性导致的对象变动，但可以使用 Vue.delete 或 vm.$delete 方法确保删除操作被检测到并触发更新视图。当需要为已有对象赋值多个新 property 时，可以使用 Object.assign() 方法将原对象与要混合进去的对象的 property 一起创建一个新的对象，从而触发更新。

在数组中，对于已经创建的实例，Vue 不能检测利用索引直接设置一个数组项，以及修改数组的长度导致的数组变动。但可以使用 Vue.set 或 vm.$set 方法，以及数组的 splice、push 等一些方法为数组添加项并触发更新视图。并且 Vue 不能检测使用 delete 操作符删除数组的某个项导致的数组变动，但可以使用 Vue.delete 或 vm.$delete 方法确保删除操作被检测到并触发更新视图。

响应式声明

Vue 不允许动态添加根级响应式 property，如果添加的根级 property 未在 data 选项中声明，Vue 将警告渲染函数正在试图访问不存在的 property。动态添加不是根级的 property 不会警告，当然它不会是响应式的。

这样的限制消除了在依赖项跟踪系统中的一类边界情况，也使 Vue 实例能更好地配合类型检查系统工作，同时也增强了代码的可维护性。所以必须在初始化实例前声明所有根级响应式 property，哪怕只是一个空值。

异步更新队列

为了避免不必要的计算和 DOM 操作，Vue 在更新 DOM 时是异步执行的。当侦听到数据变化时，Vue 将开启一个队列，并缓冲在同一事件循环中发生的所有数据变更。如果同一个 watcher 被多次触发，只会被推入到队列中一次。当同步执行栈执行完毕后，才会执行异步队列，所以在更改数据后，页面不会立刻重新渲染。

如果想基于更新后的 DOM 状态来做点什么，可以在修改数据之后立即使用 Vue.nextTick(callback)，然后等待 DOM 更新，下次 DOM 更新循环之后，会立刻执行回调函数。

Vue 在内部对异步队列依次尝试使用原生的 Promise.then、MutationObserver 和 setImmediate，如果执行环境不支持，则会采用 setTimeout(fn, 0) 代替。前两者是微任务，后两者是宏任务，微任务会在宏任务前执行，感觉上微任务会比宏任务快点。
