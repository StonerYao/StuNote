Vue 组件

在 Vue2.0 中，组件是代码复用和抽象的主要形式。

组件注册

注册全局组件

全局组件可在任何 Vue 根实例及其子组件中使用。

<!-- 在 DOM 中组件必须在通过 new Vue 创建的 Vue 根实例中作为自定义元素来使用 -->
<div id="app">
  <!-- 在 DOM 中自定义元素名称只能是短横线分隔命名 -->
  <!-- 在 DOM 中不能使用自闭合的自定义元素 -->
  <my-component></my-component>
  <!-- 组件可以任意次数的复用 -->
  <my-component></my-component>
</div>

Vue.component('my-component', Vue.extend({ /* 选项 */ }))

Vue.component('my-component', { /* 选项 */ })
// 选项对象中可接收与 new Vue 相同的选项
// 选项对象中接收的 data 选项必须为函数形式
// 选项对象中不接收 el 选项
new Vue({ el: '#app' })

注册局部组件

局部组件仅可在单个 Vue 实例中使用并且在其子组件中不可用。

<div id="app">
  <component-a></component-a>
  <component-b></component-b>
</div>

var ComponentA = { /* 选项 */ }
var ComponentB = { /* 选项 */ }
new Vue({
  el: '#app',
  components: {
    // 使用短横线分隔命名
    'component-a': ComponentA,
    // 使用大驼峰命名
    'ComponentB': ComponentB
  },
  // 在 template 中使用组件
  template: `
    <!-- 模板字符串中多个自定义元素必须使用父元素包裹 -->
    <div id="app">
      <!-- 模板字符串中可以使用自闭合的自定义元素 -->
      <component-a />
      <!-- 模板字符串中可以使用大驼峰式的自定义元素名称 -->
      <ComponentB></ComponentB>
      <!-- 模板字符串中也可以使用短横线分隔的方式引用大驼峰命名的组件 -->
      <component-b></component-b>
    </div>
  `
})

获取注册的组件

var MyComponent = Vue.component('my-component')

Prop

Prop 类型与验证

通过 props 选项向组件传递数据，当 props 为类型是对象时可进行验证。

<div id="app">
  <!-- 通过自定义 attribute 传递数据给组件 -->
  <blog-post title="My journey with Vue"></blog-post>
  <!-- 动态赋予一个变量的值传递给组件 -->
  <blog-post v-bind:title="post.title"></blog-post>
  <!-- 传递数字给组件 -->
  <blog-post v-bind:likes="42"></blog-post>
  <!-- 传递布尔值给组件 -->
  <!-- 没有值的 attribute 都意味着 true -->
  <blog-post is-published></blog-post>
  <blog-post v-bind:is-published="false"></blog-post>
  <!-- 传递数组给组件 -->
  <blog-post v-bind:comment-ids="[234, 266, 273]"></blog-post>
  <!-- 传递对象给组件 -->
  <blog-post
    v-bind:id="post.id"
    v-bind:title="post.title"
  ></blog-post>
  <!-- 传递对象所有 property 给组件 -->
  <blog-post v-bind="post"></blog-post>
</div>

Vue.component('blog-post', {
  // 注册组件可接收的 attribute 数据
  // prop 可以是任意数量的任何值
  props: ['title', 'id'],
  // 在组件实例中访问 attribute 数据如同访问 data 中的值一样
  template: '<h3 :id="id">{{ title }}</h3>'
})

Vue.component('my-component', {
  // 开发环境中当验证失败时 Vue 将会产生一个控制台的警告
  props: {
    // 基础的类型检查
    // null 和 undefined 会通过任何类型验证
    // type 可以是一个自定义的构造函数并且通过 instanceof 来进行检查确认
    propA: Number,
    // 多个可能的类型
    propB: [String, Number],
    // 必填的字符串
    propC: {
      type: String,
      required: true
    },
    // 带有默认值的数字
    propD: {
      type: Number,
      default: 100
    },
    // 带有默认值的对象
    propE: {
      type: Object,
      // 实例的 property (如 data、computed 等) 在 default 或 validator 函数中是不可用的
      // 对象或数组默认值必须从一个工厂函数获取
      default: function () {
        return { message: 'hello' }
      }
    },
    // 自定义验证函数
    propF: {
      // validator 函数接收 prop 的值作为唯一的参数代入
      validator: function (value) {
        // 若函数返回一个 falsy 的值则验证失败
        return ['success', 'warning', 'danger'].indexOf(value) !== -1
      }
    }
  }
})

单向数据流

父级 prop 的更新会向下流动到子组件中并将子组件中所有的 prop 都刷新为最新的值使得其父子 prop 之间形成了一个单向下行绑定。

// 受单向数据流限制如果在一个子组件内部改变 prop 的值 Vue 会在浏览器的控制台中发出警告
// 可以通过定义一个本地的 data property 并将这个 prop 用作其初始值
Vue.component('my-component', {
  // 传递一个初始值
  props: ['value'],
  // 将传递的值作为一个本地的 prop 数据来使用
  data: function () {
    return {
      counter: this.value
    }
  }
})
// 可以通过使用这个 prop 的值来定义一个计算属性
Vue.component('my-component', {
  // 传入原始值
  props: ['value'],
  // 将原始值进行转换后使用
  computed: {
    normalizedSize: function () {
      return this.value.trim().toLowerCase()
    }
  }
})
// 如果在子组件中改变变更一个数组或对象类型的 prop 将会影响到父组件的状态为此可以使用深度克隆

非 Prop 的 Attribute

向组件传递未被 prop 注册的特性，接受任意的 attribute 并添加到组件的根元素上，替换/合并已有的 Attribute。

<div id="app">
  <my-component
    type="text"
    class="form-base"
    label="msg"
    :value="data"
  ></my-component>
</div>

Vue.component('my-component', {
  props: ['value'],
  // 对于相同的 attribute 从组件外部提供的值会替换掉组件内部设置好的值
  // 对于 class 和 style 内外的值会被合并起来从而得到最终的值
  template: '<input type="date" class="form-control">'
  // 传入的 type="text" 会替换掉 type="date"
  // 传入的 class="form-base" 会合并成 class="form-control form-base"
  // 传入的 label="msg" 会被添加
  // 传入的 value="data" 已被注册不会添加
})
// 禁用 Attribute 继承
Vue.component('my-component', {
  // 取消组件的根元素继承 attribute 的默认行为
  // 不会影响 style 和 class 的绑定
  inheritAttrs: false,
  // 被 prop 注册的 attribute 不会继承到根元素上
  props: ['label', 'value'],
  // 通过 $attrs 获取未被 prop 注册的 attribute
  // 可以手动决定这些 attribute 被赋予的元素
  template: `
    <label>
      {{ label }}
      <input v-bind="$attrs">
    </label>
  `
})

自定义事件

监听子组件事件

<div id="app">
  <!-- 监听子组件实例的自定义事件 -->
  <welcome-button v-on:welcome="sayHi"></welcome-button>
  <!-- 通过 $event 获取子组件 $emit 抛出的值 -->
  <welcome-button v-on:welcome="welcomeText += $event"></welcome-button>
</div>

// 通过 $emit 触发实例上的自定义事件
// 自定义事件名需要使用 kebab-case 形式命名
// 事件名不存在任何自动化的大小写转换而是触发的事件名需要完全匹配监听这个事件所用的名称
Vue.component('welcome-button', {
  template: `
    <button v-on:click="$emit('welcome')">
      Click me to be welcomed
    </button>
  `
})
// 通过 $emit 触发自定义事件并抛出值
// 当父级组件监听自定义事件时可以通过 $event 访问被抛出的值
// 当自定义事件处理函数是一个方法时这个值将会作为第一个参数传入这个方法
Vue.component('welcome-button', {
  template: `
    <button v-on:click="$emit('welcome', 'hello')">
      Click me to be welcomed
    </button>
  `
})

自定义组件的双向数据绑定

<div id="app">
  <!-- 通过 v-bind 和 v-on 实现组件的双向数据绑定 -->
  <welcome-input
    v-bind:value="searchText"
    v-on:input="searchText = $event"
  ></welcome-input>
  <!-- 通过 v-model 实现组件的双向数据绑定 -->
  <welcome-input v-model="searchText"></welcome-input>
  <welcome-checkbox v-model="lovingVue"></welcome-checkbox>
</div>

// 实现组件间的双向数据绑定
// 组件内的 <input> 必须将其 value attribute 绑定到一个名叫 value 的 prop 上
// 组件内的 <input> 必须在其 input 事件被触发时将新的值通过自定义的 input 事件抛出
Vue.component('welcome-input', {
  props: ['value'],
  template: `
    <input
      v-bind:value="value"
      v-on:input="$emit('input', $event.target.value)"
    >
  `
})
// 通过 model 选项定制使用 v-model 时的 prop 和 event
Vue.component('welcome-checkbox', {
  model: {
    // 绑定 checked 属性
    prop: 'checked',
    // 监听 change 事件
    event: 'change'
  },
  props: {
    checked: Boolean
  },
  template: `
    <input
      type="checkbox"
      v-bind:checked="checked"
      v-on:change="$emit('change', $event.target.checked)"
    >
  `
})

将原生事件绑定到组件

<div id="app">
  <!-- 通过 v-on 的 .native 修饰符监听子组件实例根元素上的原生事件  -->
  <welcome-input v-on:focus.native="onFocus"></welcome-input>
  <!-- 在子组件实例中同时监听多个事件 -->
  <welcome-input v-on:focus="onFocus" v-on:welcome="sayHi"></welcome-input>
</div>

// 通过 v-on 的 .native 修饰符将原生事件绑定到子组件的根元素上
// 当根元素不支持此原生事件时监听器将失效不会报错
Vue.component('welcome-input', {
  template: `<input type="text">`
})
// 通过 $listeners 在子组件中获取父作用域中的所有事件监听器并通过 v-on 绑定在任意元素上
Vue.component('welcome-input', {
  inheritAttrs: false,
  props: ['label'],
  template: `
    <label>
      {{ label }}
      <input v-on="$listeners" />
    </label>
  `
})

使用 .sync 修饰符

<div id="app">
  <!-- 通过 v-bind 的 .sync 修饰符实现组件的双向数据绑定 -->
  <!-- 带有 .sync 修饰符的 v-bind 不能和表达式及对象字面量一起使用 -->
  <welcome-input v-bind:value.sync="searchText"></welcome-input>
  <!-- 为多个 prop 添加独立的 v-on 监听器 -->
  <welcome-input v-bind.sync="searchText"></welcome-input>
</div>

// 通过 v-bind 的 .sync 修饰符实现组件间的双向数据绑定
Vue.component('welcome-input', {
  props: ['value'],
  template: `
    <input
      :value="value"
      v-on:input="$emit('update:value', $event.target.value)"
    />
  `
})

插槽

从组件获取数据，向组件传递内容。

  • 基本使用
<my-demo>Here is the test text.</my-demo>

Vue.component("my-demo", {
  template: `
    <div>
      <strong>Hello!</strong>
      <slot></slot>
    </div>
    `,
});

如果 template 中没有包含一个 <slot> 元素，则该组件起始标签和结束标签之间的任何内容都会被抛弃。

  • 渲染洁果
<div>
  <strong>Hello!</strong>
  "Here is the test text."
</div>
  • 注意事项

编译作用域

父级模板里的所有内容都是在父级作用域中编译的；子模板里的所有内容都是在子作用域中编译的。

  • 扩展部分

后备内容

当使用组件未提供插槽时，后备内容将会被渲染。如果提供插槽，则后备内容将会被取代。

Vue.component("my-demo", {
  template: `
    <div>
      <strong>Hello!</strong>
      <slot>This will be displayed if the test text is missing.</slot>
    </div>
    `,
});

具名插槽

同时使用多个不同的插槽。

<my-demo>
  <template v-slot:header>
    <h1>Here might be a page title</h1>
  </template>

  <p>A paragraph for the main content.</p>
  <p>And another one.</p>

  <template v-slot:footer>
    <p>Here's some contact info</p>
  </template>
</my-demo>

或

<my-demo>
  <template v-slot:header>
    <h1>Here might be a page title</h1>
  </template>

  <template v-slot:default>
    <p>A paragraph for the main content.</p>
    <p>And another one.</p>
  </template>

  <template v-slot:footer>
    <p>Here's some contact info</p>
  </template>
</my-demo>

Vue.component("my-demo", {
  template: `
    <div class="container">
      <header>
        <slot name="header"></slot>
      </header>
      <main>
        <slot></slot>
      </main>
      <footer>
        <slot name="footer"></slot>
      </footer>
    </div>
  `,
});

具名插槽的缩写

<my-demo>
  <template #header>
    <h1>Here might be a page title</h1>
  </template>

  <p>A paragraph for the main content.</p>
  <p>And another one.</p>

  <template #footer>
    <p>Here's some contact info</p>
  </template>
</my-demo>

一个不带 name 的 <slot> 出口会带有隐含的名字“default”。
v-slot 只能添加在 <template> 上，除了独占默认插槽的缩写语法。
具名插槽的缩写只能用在其有参数的时候。

作用域插槽

让插槽内容能够访问子组件中的数据。

<my-demo>
  <template v-slot:default="slotProps">
    {{ slotProps.user.firstName }}
  </template>
</my-demo>

Vue.component("my-demo", {
  data: function () {
    return {
      user: {
        firstName: "Michael",
        lastName: "Jackson"
      }
    };
  },
  template: `
    <span>
      <slot v-bind:user="user">
        {{ user.lastName }}
      </slot>
    </span>
  `,
});

独占默认插槽的缩写语法

<my-demo v-slot:default="slotProps">
  {{ slotProps.user.firstName }}
</my-demo>

或

<my-demo v-slot="slotProps">
  {{ slotProps.user.firstName }}
</my-demo>

默认插槽的缩写语法只适用于只有默认插槽时。它不能和具名插槽混用，因为它会导致作用域不明确。

解构插槽 Prop

<my-demo v-slot:default="{ user }">
  {{ user.firstName }}
</my-demo>

解构时对 prop 重命名以及定义后备内容同样适用于此。

动态插槽名

在 v-slot 上使用动态指令参数。

<my-demo>
  <template v-slot:[dynamicSlotName]>
    <p>Here is the test text.</p>
  </template>
</my-demo>

动态组件

使不同组件之间进行动态切换。

  • 基本使用
<component v-bind:is="tab-home"></component>

Vue.component("tab-home", {
  template: "<div>Home component</div>"
});
Vue.component("tab-posts", {
  template: "<div>Posts component</div>"
});
Vue.component("tab-archive", {
  template: "<div>Archive component</div>"
});

或

<component v-bind:is="tabs[0].component"></component>

var tabs = [
  {
    name: "Home",
    component: {
      template: "<div>Home component</div>"
    }
  },
  {
    name: "Posts",
    component: {
      template: "<div>Posts component</div>"
    }
  },
  {
    name: "Archive",
    component: {
      template: "<div>Archive component</div>"
    }
  }
];
  • 注意事项

每一次切换都会销毁旧组件实例并重新创建动态组件。

  • 扩展部分

在动态组件上使用 keep-alive

缓存不活动的组件实例。

<keep-alive>
  <component v-bind:is="tab-home"></component>
</keep-alive>

<keep-alive> 要求被切换到的组件都要有自己的名字，以用于 include 和 exclude 的匹配。
<keep-alive> 是一个抽象组件：它自身不会渲染一个 DOM 元素，也不会出现在组件的父组件链中。
当组件在 <keep-alive> 内被切换，它的以及树内的所有嵌套组件的 activated 和 deactivated 这两个生命周期钩子函数将会被对应执行。
<keep-alive> 要求同时只有一个子元素被渲染，用于其一个直属的子组件被开关的情形。
<keep-alive> 不会在函数式组件中正常工作，因为它们没有缓存实例。

异步组件

只在需要的时候才从服务器加载组件。

  • 基本使用
// 局部注册
new Vue({
  // ...
  components: {
    'my-component': () => import('./my-async-component')
  }
})

// 全局注册
Vue.component(
  'my-component',
  // 这个动态导入会返回一个 `Promise` 对象。
  () => import('./my-async-component')
)

可以将多个需要同时加载的组件合并到一个文件中。

new Vue({
  // ...
  components: {
    'AsyncCmp1': () => import(/* webpackChunkName: "async" */ './AsyncCmp1'),
    'AsyncCmp2': () => import(/* webpackChunkName: "async" */ './AsyncCmp2')
  }
})
  • 注意事项

异步加载的组件，会在 <link> 上设置 rel="prefech"。浏览器会在空闲时间内下载对应的资源，使用时可以直接从缓存中获取。与之对应的 rel="preload"，会及时下载对应的资源。

  • 扩展部分

处理加载状态

异步组件工厂函数也可以返回一个如下格式的对象：

const AsyncComponent = () => ({
  // 需要加载的组件 (应该是一个 `Promise` 对象)
  component: import('./MyComponent.vue'),
  // 异步组件加载时使用的组件
  loading: LoadingComponent,
  // 加载失败时使用的组件
  error: ErrorComponent,
  // 展示加载时组件的延时时间。默认值是 200 (毫秒)
  delay: 200,
  // 如果提供了超时时间且组件加载也超时了，
  // 则使用加载失败时使用的组件。默认值是：`Infinity`
  timeout: 3000
})

处理边界情况

处理需要对 Vue 的规则做一些小调整的特殊情况有关的功能。

需要注意这些功能都是有劣势或危险的场景的。

访问元素 & 组件

访问根实例

  • 基本使用
new Vue({
  data: {
    foo: 1
  },
  computed: {
    bar: function () { /* ... */ }
  },
  methods: {
    baz: function () { /* ... */ }
  }
})

// 获取根组件的数据
this.$root.foo

// 写入根组件的数据
this.$root.foo = 2

// 访问根组件的计算属性
this.$root.bar

// 调用根组件的方法
this.$root.baz()
  • 注意事项

访问根实例难以维护，推荐使用 Vuex 来管理应用的状态。

访问父级组件实例

  • 基本使用
Vue.component("demo-parent", {
  data: function() {
    return {
      foo: 1
    }
  }
})

Vue.component("demo-child", {
  created: function() {
    var fooNew = this.$parent.foo;
  }
})
  • 注意事项

访问父级组件实例难以维护，且在深层级的组件上使用时容易失控。

访问子组件实例或子元素

  • 基本使用
<div id="app">
  <my-demo ref="myComponent"></my-demo>
  <input ref="myElement" />
</div>

new Vue({
  el: "#app",
  mounted: function() {
    var component = this.$refs.myComponent;
    var element = this.$refs.myElement;
    element.focus();
  }
})
  • 注意事项

访问子组件实例或子元素只会在组件渲染完成之后生效，并且它们不是响应式的。

  • 扩展部分

当 ref 和 v-for 一起使用的时候，得到的 ref 将会是一个包含了对应数据源的这些子组件的数组。

依赖注入

指定要提供给后代组件的数据/方法，然后在后代组件中接收这些数据/方法。

  • 基本使用
Vue.component("demo-parent", {
  provide: function () {
    return {
      foo: 1
    }
  }
})

Vue.component("demo-child", {
  inject: ['foo']
})
  • 注意事项

依赖注入难以维护，并且它们不是响应式的，推荐使用 Vuex 来管理应用的状态。

程序化的事件侦听器

利用 Vue 实例中的事件监听方法手动侦听事件。

通过 $on(eventName, eventHandler) 侦听一个事件
通过 $once(eventName, eventHandler) 一次性侦听一个事件
通过 $off(eventName, eventHandler) 停止侦听一个事件

Vue 的事件系统不同于浏览器的 EventTarget API，尽管它们工作起来是相似的。

  • 基本使用
new Vue({
  mounted: function() {
    var picker = new Pikaday();

    this.$once("hook:beforeDestroy", function() {
      picker.destroy();
    });
  }
});
  • 注意事项

程序化的事件侦听器可以用于代码组织工具，不过最好的方式通常还是创建更多可复用的模块化组件。

循环引用

递归组件

组件通过 name 选项在其模板中调用自身。

当使用 Vue.component 全局注册一个组件时，这个全局的 ID 会自动设置为该组件的 name 选项。

  • 基本使用
name: 'stack-overflow',
template: '<div><stack-overflow></stack-overflow></div>'
  • 注意事项

递归组件可能导致无限循环，需要确保递归调用是条件性的。

组件之间的循环引用

组件在其模板中相互调用。

  • 基本使用
name: 'tree-folder',
template: '<div><tree-folder-contents></tree-folder-contents></div>'

name: 'tree-folder-contents',
template: '<div><tree-folder></tree-folder></div>'
  • 注意事项

组件之间的循环引用可能导致在使用打包工具模块系统依赖/导入组件时报错，为此需要等到生命周期钩子 beforeCreate 函数执行时再依赖/导入，或在本地注册组件时使用打包工具的异步依赖/导入。通过 Vue.component 全局注册的组件不存在以上问题。

模板定义的替代品

内联模板

组件通过 inline-template 特性将其里面的内容作为模板。

  • 基本使用
<div id='app'>
  <my-component inline-template>
    <p>Hello hello hello</p>
  </my-component>
</div>
  • 注意事项

内联模板会让模板的作用域变得更加难以理解，推荐使用 template 选项或 .vue 文件里的 <template> 元素来定义模板。

X-Template

通过 <script> 元素的 text/x-template 类型定义模板。

  • 基本使用
<div id='app'>
  <my-component></my-component>
</div>
<script type="text/x-template" id="my-component-template">
  <p>Hello hello hello</p>
</script>

Vue.component('my-component', {
  template: '#my-component-template'
})
  • 注意事项

X-Template 会将模板和该组件的其它定义分离开，难以维护。

控制更新

强制更新

  • 基本使用
Vue.component('my-component', {
  data: function() {
    return {
      userInfo: {name: 'zhangsan'}
    }
  },
  methods: {
    updateName() {
      this.userInfo.name = 'lisi';
      this.$forceUpdate();
    }
  }
  • 注意事项

强制更新仅仅影响实例本身和插入插槽内容的子组件。通常 Vue 的响应式系统会进行自动更新，只有在极少数的情况下需要手动强制更新。

通过 v-once 创建低开销的静态组件

对于包含了大量静态内容的组件，为避免渲染时重复的计算，而通过 v-once 将计算结果缓存起来。

  • 基本使用
Vue.component('terms-of-service', {
  template: `
    <div v-once>
      <h1>Terms of Service</h1>
      ... a lot of static content ...
    </div>
  `
})
  • 注意事项

通过 v-once 创建低开销的静态组件可能导致无法正确更新，难以维护。

eventBus(事件总线)

  • 基本使用
<div id="app">
  <my-cmp></my-cmp>
  <my-cmp1></my-cmp1>
</div>

Vue.prototype.$bus = new Vue();

Vue.component('my-cmp', {
  data () {
    return {
      name: 'my-cmp',
    }
  },
  methods: {
    onClick () {
      this.$bus.$emit('bus', this.name);
    }
  },
  template: `
    <div @click="onClick">my-cmp</div>
  `
})

Vue.component('my-cmp1', {
  mounted () {
    this.$bus.$on('bus', (val) => {
      console.log(val);
    })
  },
  template: `
    <div>my-cmp1</div>
  `
})
  • 注意事项

eventBus(事件总线)会造成代码混乱，难以维护。
