Axios

axios 是一个基于 promise 的 HTTP 库，可以用在浏览器和 node.js 中。

axios 可使用 cdn 或 npm 安装 $ npm install axios。

创建请求

通过向 axios 传递相关配置来创建请求。

  • axios(config)
  • axios(url, [config])
axios({
    // 配置请求
    method: 'post',
    url: '/user/12345',
    data: {
      firstName: 'Fred',
      lastName: 'Flintstone'
    }
  })
  .then(function (response) {
    // 请求成功
  })
  .catch(function (error) {
    // 请求失败
  })
  .then(function () {
    // 始终执行
  });

配合 async/await 创建请求。

async function getUser() {
  try {
    const response = await axios.get('/user?ID=12345');
    console.log(response);
  } catch (error) {
    console.error(error);
  }
};

创建实例

使用自定义配置新建一个 axios 实例。

axios.create([config])

const instance = axios.create({
  baseURL: 'https://api.example.com'
});

请求方法别名创建请求

  • {axios | instance}.request(config)
  • {axios | instance}.get(url, [config])
  • {axios | instance}.post(url, [data], [config]])
  • {axios | instance}.delete(url, [config])
  • {axios | instance}.head(url, [config])
  • {axios | instance}.put(url, [data], [config])
  • {axios | instance}.patch(url, [data], [config]])
  • {axios | instance}.options(url, [config])
instance.get('/getUserInfo', {
  timeout: 5000
});

创建并发请求

同时进行多个请求，并统一处理返回值。

  • axios.all(iterable)
  • axios.spread(callback)
axios.all([
  axios.get('/a'),
  axios.get('/b')
]).then(axios.spread((aRes, bRes) => {
  console.log(aRes, bRes);
}));

使用 Promise.all 执行多个并发请求。

function getUserAccount() {
  return axios.get('/user/12345');
}

function getUserPermissions() {
  return axios.get('/user/12345/permissions');
}

Promise.all([getUserAccount(), getUserPermissions()])
  .then(function (results) {
    const acct = results[0];
    const perm = results[1];
  });

创建默认配置

指定将被用在各个请求的默认配置。配置会以全局的默认配置，然后是实例的默认配置，最后是实例自身的配置的优先顺序进行合并。对于相同的 options 后者优先于前者。

  • axios.defaults.[options] 全局的默认配置
  • instance.defaults.[options] 实例的默认配置
axios.defaults.baseURL = 'https://api.example.com';
instance.defaults.timeout = 2500;

常用请求配置

创建请求时的配置选项。只有 url 是必需的。

  • url {string}: 请求的服务器 URL 路径。
  • method {string}: 创建请求时使用的方法，默认 get，如需适配 jsonp 需要用 axios-jsonp 插件。
  • baseURL {string}: 请求的域名和基础路径，将自动加在 url 前面。
  • params {object}: 与请求一起发送的 URL 参数，用于 get 方法。
  • data {object}: 请求主体被发送的数据，用于 post 方法。
  • headers {object}: 被发送的自定义请求头。
  • timeout {number}: 请求超时的毫秒数。

响应结构

  • data {object}: 响应数据。
  • status {number}: HTTP 状态码。
  • statusText {string}: HTTP 状态信息。
  • headers {object}: 响应头。
  • config {object}: 请求配置。
  • request {object}: 请求。
axios.get('/user/12345')
  .then(function (response) {
    console.log(response.data);
    console.log(response.status);
    console.log(response.statusText);
    console.log(response.headers);
    console.log(response.config);
  });

拦截器

在发起请求之前做一些处理，或者在响应回来之后做一些处理。

  • axios.interceptors.request.use() 设置请求拦截
  • axios.interceptors.response.use() 设置响应拦截
  • axios.interceptors.request.eject() 取消请求拦截
  • axios.interceptors.response.eject() 取消响应拦截
// 添加请求拦截器
axios.interceptors.request.use(function (config) {
    // 在发送请求之前做些什么，可通过 config 参数获得请求配置
    return config;
  }, function (error) {
    // 对请求错误做些什么
    return Promise.reject(error);
  });

// 添加响应拦截器
axios.interceptors.response.use(function (response) {
    // 对响应数据做点什么，可通过 response 参数获得响应
    return response;
  }, function (error) {
    // 对响应错误做点什么
    return Promise.reject(error);
  });

// 移除拦截器
const myInterceptor = axios.interceptors.request.use(function () {/*...*/});
axios.interceptors.request.eject(myInterceptor);

为自定义 axios 实例添加拦截器。

const instance = axios.create();
instance.interceptors.request.use(function () {/*...*/});

错误处理

axios.get('/user/12345')
  .catch(function (error) {
    if (error.response) {
      // 已发出请求，但服务器响应的状态码超出了2xx的范围
      console.log(error.response.data);
      console.log(error.response.status);
      console.log(error.response.headers);
    } else if (error.request) {
      // 已发出请求，但未得到服务器的响应
      console.log(error.request);
    } else {
      // 在发送请求的过程中取消请求，引发了错误
      console.log('Error', error.message);
    }
    console.log(error.config);
  });

可以使用 validateStatus 配置选项定义一个自定义 HTTP 状态码的错误范围。

axios.get('/user/12345', {
  validateStatus: function (status) {
    // 只有当状态码小于500时才会处理
    return status < 500;
  }
});

使用 toJSON 可以得到一个包含更多 HTTP 错误信息的对象。

axios.get('/user/12345')
  .catch(function (error) {
    console.log(error.toJSON());
  });

取消请求

Axios 的 cancel-token API 基于 cancelable-promises 提案，它可以使用同一个 cancel-token 取消多个请求。

const CancelToken = axios.CancelToken;
// 使用 CancelToken.source 工厂方法创建 cancel-token
const source = CancelToken.source();

axios.get('/user/12345', {
  cancelToken: source.token
}).catch(function(thrown) {
  if (axios.isCancel(thrown)) {
    console.log('Request canceled', thrown.message);
  } else {
     // 处理错误
  }
});

axios.post('/user/12345', {
  name: 'new name'
}, {
  cancelToken: source.token
});

// 取消请求（message 参数是可选的）
source.cancel('Operation canceled by the user.');

还可以通过传递一个 executor 函数到 CancelToken 的构造函数来创建 cancel-token。

const CancelToken = axios.CancelToken;
let cancel;

axios.get('/user/12345', {
  cancelToken: new CancelToken(function executor(c) {
    // executor 函数接收一个 cancel 函数作为参数
    cancel = c;
  })
});

// 取消请求
cancel();

浏览器预检

当 axios 的请求为非简单请求时，浏览器会进行预检，及发送 options 请求。请求到服务器，询问是否允许跨域。如果响应中允许预检中请求的跨域行为，则浏览器会进行真正的请求。否则会报 405 错误。
