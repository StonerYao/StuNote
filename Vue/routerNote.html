Vue Router

Vue Router 是 Vue 官方的路由管理器。多用于创建单页应用。让页面实现非重载就能刷新数据。

安装

  • 直接下载 / CDN: https://unpkg.com/vue-router/dist/vue-router.js
  • NPM: npm install vue-router
  • Vue CLI: vue add router

基本使用

  • JavaScript

    1. 引入路由

      <!-- 直接下载 / CDN -->
<script src="/path/to/vue.js"></script>
<script src="/path/to/vue-router.js"></script>

      // NPM + 模块化
import Vue from 'vue'
import VueRouter from 'vue-router'

Vue.use(VueRouter)

    2. 定义 (路由) 组件

      const Foo = { template: '<div>foo</div>' }
const Bar = { template: '<div>bar</div>' }

    3. 定义路由

      const routes = [
  { path: '/foo', component: Foo },
  { path: '/bar', component: Bar }
]

    4. 创建 router 实例

      const router = new VueRouter({
  routes
})

    5. 创建和挂载根实例

      const app = new Vue({
  router
}).$mount('#app')

  • HTML

    1. 使用 router-link 组件来导航

      <router-link to="/foo">Go to Foo</router-link>
<router-link to="/bar">Go to Bar</router-link>

    2. 使用 router-view 组件来渲染

      <router-view></router-view>

路由模式

路由模式由 mode 选项配置。

const router = new VueRouter({
  mode: 'history',
  routes: [...]
})
  • hash: 使用 URL hash 值。当 URL 改变时，页面不会重新加载。
  • history: 依赖 HTML5 History API 和服务器配置。当 URL 改变时，页面不会重新加载。
  • abstract: 如果发现没有浏览器的 API，路由会自动强制进入这个模式。用于 Node.js 服务器端。

注意: 使用 history 模式时，如果后台没有正确的配置，当从浏览器直接访问一个匹配不到任何静态资源的 URL 就会返回 404。所以要在服务端增加一个覆盖所有情况的候选资源。或者可以用服务端路由匹配到来的 URL，并在没有匹配到路由的时候返回 404，以实现回退。

命名路由

命名路由由 routes 配置中的 name 选项配置。

const router = new VueRouter({
  routes: [
    {
      path: '/user/:userId',
      name: 'user',
      component: User
    }
  ]
})

<!-- 把路由导航到 /user/123 路径 -->
<router-link :to="{ name: 'user', params: { userId: 123 }}">User</router-link>

或

// 把路由导航到 /user/123 路径
router.push({ name: 'user', params: { userId: 123 }})

当路由较为复杂时，通过命名路由链接或跳转，会更方便一些。

嵌套路由

嵌套路由由 routes 配置中的 children 选项配置。

const router = new VueRouter({
  routes: [
    { path: '/user/:id',
      component: User,
      children: [
        {
          // 当 /user/:id 匹配成功
          // UserHome 会被渲染在 User 的 <router-view> 中
          path: '',
          component: UserHome
        },
        {
          // 当 /user/:id/posts 匹配成功
          // UserPosts 会被渲染在 User 的 <router-view> 中
          path: 'posts',
          component: UserPosts
        }
      ]
    }
  ]
})

注意: 以 / 开头的嵌套路径会被当作根路径处理，从而使用嵌套组件而无须设置嵌套的路径。

const User = {
  template: `
    <div class="user">
      <h2>User {{ $route.params.id }}</h2>
      <router-view></router-view>
    </div>
  `
}

通过嵌套路由可以在应用页面上实现多层组件的嵌套。

重定向和别名

重定向由 routes 配置中的 redirect 选项配置。

const router = new VueRouter({
  routes: [
    { path: '/a', redirect: '/b' }
    { path: '/b', redirect: { name: 'foo' }}
    { path: '/c', redirect: to => {
      // 方法接收 目标路由 作为参数
      // return 重定向的 字符串路径/路径对象
    }}
  ]
})

注意: 导航守卫并没有应用在跳转路由上，而仅仅应用在其目标上。

通过重定向可以使一个路由跳转到另一个路由上。

别名由 routes 配置中的 alias 选项配置。

const router = new VueRouter({
  routes: [
    { path: '/a', component: A, alias: '/b' }
  ]
})

通过别名可以使不同的路由对应一个路由。从而可以自由地将 UI 结构映射到任意的 URL。

编程式的导航

借助 router 的实例方法，通过编写代码来实现导航。

注意: 在 Vue 实例内部，可以通过 $router 访问路由实例。因此可以调用 this.$router.push。

点击 <router-link :to="..."> 等同于调用 router.push(...)。
点击 <router-link :to="..." replace> 等同于调用 router.replace(...)。

router.push 或 router.replace 方法的参数可以是一个字符串路径，或者一个描述地址的对象。同样的规则也适用于 router-link 组件的 to 属性。

// 字符串
router.push('home')

// 对象
router.push({ path: 'home' })

// 带查询参数 => /register?plan=private
router.push({ path: 'register', query: { plan: 'private' }})

// 命名的路由 => /user/123
const userId = '123'
router.push({ name: 'user', params: { userId }})

// 使用带有参数的 path 替代 params => /user/123
const userId = '123'
router.push({ path: `/user/${userId}` })

注意: 如果提供了 path，params 会被忽略，但 query 不会被忽略。可以通过提供 name 或完整的带有参数的 path 来解决。

在 router.push 或 router.replace 中提供 onComplete 和 onAbort 回调作为第二个和第三个参数。这些回调将会在导航成功完成 (在所有的异步钩子被解析之后) 或终止 (导航到相同的路由、或在当前导航完成之前导航到另一个不同的路由) 的时候进行相应的调用。

router.push(location, onComplete?, onAbort?)
router.replace(location, onComplete?, onAbort?)

可以省略第二个和第三个参数，此时如果支持 Promise，router.push 或 router.replace 将返回一个 Promise。

router.push(location).then(onComplete).catch(onAbort)
router.replace(location).then(onComplete).catch(onAbort)

注意: 如果目的地和当前路由相同，只有参数发生了改变 (比如 /users/1 -> /users/2)，则需要使用 beforeRouteUpdate 来响应这个变化。

router.go 方法的参数是一个整数。

// 在浏览器记录中前进一步 等同于 router.forward()
router.go(1)

// 后退一步记录 等同于 router.back()
router.go(-1)

// 前进 3 步记录
router.go(3)

// 仅刷新
router.go(0)

// 如果 history 记录不够用就默默地失败
router.go(-100)
router.go(100)

Vue Router 的导航方法 (push、 replace、 go) 在各类路由模式 (history、 hash 和 abstract) 下表现一致。

动态路由匹配

在 vue-router 的路由路径中使用动态路径参数，以达到动态路由的效果。

const User = {
  template: '<div>User</div>'
}

const router = new VueRouter({
  routes: [
    // 此时 /user/foo 和 /user/bar 都将映射到相同的路由
    { path: '/user/:id', component: User }
  ]
})

一个动态路径参数使用冒号 : 标记。当匹配到一个路由时，参数值会被设置到 this.$route.params，可以在每个组件内使用。

const User = {
  template: '<div>User {{ this.$route.params.id }}</div>'
}

可以在一个路由中设置多段动态路径参数，对应的值都会设置到 $route.params 中。

模式 匹配路径 $route.params
/user/:username /user/evan { username: 'evan' }
/user/:username/post/:post_id /user/evan/post/123 { username: 'evan', post_id: '123' }

注意：在使用动态路由匹配时，例如从 /user/foo 导航到 /user/bar，两个路由都渲染同个组件，为了提高效率，Vue 会复用原来的组件实例。这也意味着组件的生命周期钩子不会再被调用。为了在复用组件时，响应路由参数的变化，可以使用 watch 侦听 $route 对象，也可以使用 beforeRouteUpdate 导航守卫。

使用通配符 * 匹配任意路径。

{
  // 会匹配所有路径
  path: '*'
}
{
  // 会匹配以 `/user-` 开头的任意路径
  path: '/user-*'
}

当使用一个通配符时，$route.params 内会自动添加一个名为 pathMatch 参数。它包含了 URL 通过通配符被匹配的部分。

// 给出一个路由 { path: '/user-*' }
this.$router.push('/user-admin')
this.$route.params.pathMatch // 'admin'
// 给出一个路由 { path: '*' }
this.$router.push('/non-existing')
this.$route.params.pathMatch // '/non-existing'

注意：当同一个路径可以匹配多个路由时，匹配的优先级将按照路由的定义顺序，谁先定义的谁的优先级就最高。

一般将含有通配符的路由放在最后，例如路由 { path: '*' }，用于客户端 404 错误。

通过动态路由匹配可以把某种模式匹配到的所有路由，全都映射到同个组件。

命名视图

命名视图由 router-view 组件中的 name 属性配置。

可以在界面中拥有多个单独命名的视图，而不是只有一个单独的出口。

<router-view class="view one"></router-view>
<router-view class="view two" name="a"></router-view>
<router-view class="view three" name="b"></router-view>

一个视图使用一个组件渲染，因此对于同个路由，多个视图就需要多个组件。

const router = new VueRouter({
  routes: [
    {
      path: '/',
      components: {
        default: Foo,
        a: Bar,
        b: Baz
      }
    }
  ]
})

也可以将命名视图配合嵌套路由一起使用创建复杂的布局，从而让对应的路径显示不同的命名视图。

注意: 如果 router-view 没有设置名字，那么默认为 default。

通过命名视图可以同时 (同级) 展示多个视图，而不是嵌套展示。

路由组件传参

路由组件传参由 routes 配置中的 props 选项配置。

props 选项可以是以下几种形式:

  • 布尔模式
    当 props 被设置为 true，route.params 将会被设置为组件属性。
  • 对象模式
    当 props 被设置为对象，将会按原样设置为组件属性。
  • 函数模式
    当 props 被设置为函数，函数将接收一个 $route 作为参数，并将返回值作为组件属性。

注意: 请尽可能保持 props 函数为无状态的，因为它只会在路由发生变化时起作用。如果需要状态来定义 props，请使用包装组件，这样 Vue 才可以对状态变化做出反应。

在组件中使用 $route 会使之与其对应路由形成高度耦合，从而使组件只能在某些特定的 URL 上使用，限制了其灵活性。

const User = {
  template: '<div>User {{ $route.params.id }}</div>'
}
const router = new VueRouter({
  routes: [
    { path: '/user/:id', component: User }
  ]
})

使用 props 将组件和路由解耦。

const User = {
  props: ['id'],
  template: '<div>User {{ id }}</div>'
}
const router = new VueRouter({
  routes: [
    // 布尔模式
    { path: '/user/:id', component: User, props: true }

    // 对象模式
    { path: '/user/:id', component: User, props: { default: 'hello' } }

    // 函数模式
    { path: '/user/:id', component: User, props: (route) => ({ query: route.query.q }) }
  ]
})

通过向路由组件传参可以在任何地方使用同个组件，使得组件更易于重用和测试。

导航守卫

在路由发生改变过程中有三种方式 (全局的, 单个路由独享的, 或者组件级的) 植入守卫或钩子函数。

每个守卫方法接收三个参数:

  • to: Route: 即将要进入的目标路由。
  • from: Route: 当前导航正要离开的路由。
  • next: Function: 用来结束这个钩子的方法，并依赖其参数决定执行效果。

next 方法可接收参数:

  • next(): 进行管道中的下一个钩子。如果全部钩子执行完了，则导航的状态就是 confirmed (确认的)。
  • next(false): 中断当前的导航。如果浏览器的 URL 改变了 (可能是用户手动或者浏览器后退按钮)，那么 URL 地址会重置到 from 路由对应的地址。
  • next('/') 或者 next({ path: '/' }): 跳转到一个不同的地址。当前的导航被中断，然后进行一个新的导航。你可以向 next 传递任意位置对象，且允许设置诸如 replace: true、name: 'home' 之类的选项以及任何用在 router-link 的 to prop 或 router.push 中的选项。
  • next(error): 如果传入 next 的参数是一个 Error 实例，则导航会被终止且该错误会被传递给 router.onError() 注册过的回调。

注意: 要确保 next 函数在任何给定的导航守卫中都被严格调用一次，否则无法继续执行下一个导航守卫，从而让路由跳转停止。它可以出现多于一次，但是只能在所有的逻辑路径都不重叠的情况下，否则钩子永远都不会被解析或报错。

全局前置守卫: router.beforeEach

当一个导航触发时，全局前置守卫按照创建顺序调用。守卫是异步解析执行，此时导航在所有守卫处理完之前一直处于等待中。

const router = new VueRouter({ ... })

router.beforeEach((to, from, next) => {
  // ...
})

全局解析守卫: router.beforeResolve

在导航被确认之前，同时在所有组件内守卫和异步路由组件被解析之后，解析守卫被调用。

router.beforeResolve((to, from, next) => {
  // ...
})

全局后置钩子: router.afterEach

全局后置钩子和守卫不同，不会接收 next 函数，也不会改变导航本身。

router.afterEach((to, from) => {
  // ...
})

路由独享的守卫: beforeEnter

const router = new VueRouter({
  routes: [
    {
      path: '/foo',
      component: Foo,
      beforeEnter: (to, from, next) => {
        // ...
      }
    }
  ]
})

组件内的守卫: beforeRouteEnter

在渲染该组件的对应路由被确认前调用。此时不能访问组件实例 this，因为在守卫执行前，组件实例还没被创建。不过可以通过传一个回调给 next 来访问组件实例，在导航被确认的时候执行回调，并且把组件实例作为回调方法的参数。

const Foo = {
  template: `...`,
  beforeRouteEnter (to, from, next) {
    next(vm => {
      // 通过 `vm` 访问组件实例 在 mounted 钩子后调用
    })
  }
}

组件内的守卫: beforeRouteUpdate

在当前路由改变，但是该组件被复用时调用。动态路由跳转或路由查询变更，都可让组件进行复用。此时可以访问组件实例 this。

beforeRouteUpdate (to, from, next) {
  // ...
}

组件内的守卫: beforeRouteLeave

在导航离开该组件的对应路由时调用。此时可以访问组件实例 this。

beforeRouteLeave (to, from, next) {
  // ...
}

完整的导航解析流程:

  1. 导航被触发。
  2. 在失活的组件里调用 beforeRouteLeave 守卫。
  3. 调用全局的 beforeEach 守卫。
  4. 在重用的组件里调用 beforeRouteUpdate 守卫。
  5. 在路由配置里调用 beforeEnter。
  6. 解析异步路由组件。
  7. 在被激活的组件里调用 beforeRouteEnter。
  8. 调用全局的 beforeResolve 守卫。
  9. 导航被确认。
  10. 调用全局的 afterEach 钩子。
  11. 触发 DOM 更新。
  12. 调用 beforeRouteEnter 守卫中传给 next 的回调函数，创建好的组件实例会作为回调函数的参数传入。

注意: 参数或查询的改变并不会触发进入/离开的导航守卫。可以通过侦听 $route 对象来应对这些变化，或使用 beforeRouteUpdate 的组件内守卫。

通过导航守卫可以控制导航的通过跳转或取消，也可在导航完成前获取数据。

路由元信息

路由元信息由 routes 配置中的 meta 选项配置。

const router = new VueRouter({
  routes: [
    {
      path: '/foo',
      component: Foo,
      meta: { requiresAuth: true }
    }
  ]
})

使用 $route.meta 访问当前路由的元信息。对于嵌套路由，可以遍历 $route 对象的 $route.matched 数组中的路由记录，访问 meta 字段。

router.beforeEach((to, from, next) => {
  if (to.matched.some(record => record.meta.requiresAuth)) {
    next(false)
  } else {
    next()
  }
})

通过路由元信息可以在导航守卫中检查元字段，并根据其值配置相应的导航处理逻辑。

过渡动效

实现所有路由的过渡动效需要在 router-view 组件外配合 transition 组件一起使用。

<transition>
  <router-view></router-view>
</transition>

实现单个路由的过渡动效，可以在各路由组件内使用 <transition> 并设置不同的 name。

通过过渡动效可以实现所有路由的统一过渡，也可以配合 <transition> 的 name attribute 实现基于路由的动态过渡。

滚动行为

滚动行为由 scrollBehavior 选项配置。

const router = new VueRouter({
  routes: [...],
  scrollBehavior (to, from, savedPosition) {
    // return 期望滚动到哪个的位置
  }
})

scrollBehavior 方法接收 to 和 from 路由对象。第三个参数 savedPosition 当且仅当 popstate 导航 (通过浏览器的 前进/后退 按钮触发) 时才可用。

返回滚动位置的对象可以是以下的几种情况:

  • { x: number, y: number }
  • { selector: string, offset? : { x: number, y: number }}
  • savedPosition
  • falsy 值
  • Promise
scrollBehavior (to, from, savedPosition) {
  return { x: 0, y: 0 }; // 滚动到页面顶部
  return { selector: to.hash }; // 滚动到页面锚点处
  return savedPosition; // 滚动到页面之前的位置
  return undefined; // 不滚动
  return new Promise(resolve => {
    if (to.matched.some(m => m.meta.scrollToTop)) {
      resolve({ x: 0, y: 0 })
    }
  }) // 配合路由元信息更细颗粒度地控制滚动
}

注意: 这个功能只在支持 history.pushState 的浏览器中可用。

通过滚动行为可以自定义路由切换时页面如何滚动 (滚到顶部或者是保持原先的滚动位置)。

路由对象

路由对象表示当前激活的路由的状态信息，包含了当前 URL 解析得到的信息，还有 URL 匹配到的路由记录。路由对象是只读的，每次成功的导航后都会产生一个新的对象。

路由对象出现在多个地方:

  • 在组件内，即 this.$route。
  • 在 $route 观察者回调内。
  • router.match(location) 的返回值。
  • 导航守卫的参数。
  • scrollBehavior 方法的参数。

属性

  • $route.path
    类型: string
    对应当前路由的路径，总是解析为绝对路径。

  • $route.params
    类型: Object
    包含了动态片段和全匹配片段，如果没有路由参数，就是一个空对象。

  • $route.query
    类型: Object
    表示 URL 查询参数，如果没有查询参数，则是个空对象。

  • $route.hash
    类型: string
    当前路由的 hash 值 (带 #)，如果没有 hash 值，则为空字符串。

  • $route.fullPath
    类型: string
    完成解析后的 URL，包含查询参数和 hash 的完整路径。

  • $route.matched
    类型: Array<RouteRecord>
    包含当前路由的所有嵌套路径片段的路由记录。路由记录就是 routes 配置数组 (包含 children 数组) 中的对象副本。

  • $route.name
    类型: string
    当前路由的名称，如果有的话。

  • $route.redirectedFrom
    类型: string
    如果存在重定向，即为重定向来源的路由的名字。

构建选项

  • routes
    类型: Array<RouteConfig>
    定义路由，每个路由应该映射一个组件。
    RouteConfig 的类型定义:

    interface RouteConfig = {
  path: string, // 路由
  component?: Component, // 组件
  name?: string, // 命名路由
  components?: { [name: string]: Component }, // 命名视图组件
  redirect?: string | Location | Function, // 重定向
  props?: boolean | Object | Function, // 路由组件传参
  alias?: string | Array<string>, // 别名
  children?: Array<RouteConfig>, // 嵌套路由
  beforeEnter?: (to: Route, from: Route, next: Function) => void, // 路由独享的守卫
  meta?: any, // 路由元信息

  // 2.6.0+
  caseSensitive?: boolean, // 匹配规则是否大小写敏感？(默认值: false)
  pathToRegexpOptions?: Object // 编译正则的选项
}

  • mode
    类型: string

<router-link>

<router-link> 组件用于在具有路由功能的应用中 (点击) 导航。

通过 to 属性指定目标地址，默认渲染成带有正确链接的 <a> 标签，可以通过配置 tag 属性生成别的标签。另外，当目标路由成功激活时，链接元素自动设置一个表示激活的 CSS 类名。

<router-link> 和 <a> 的区别？

无论是 HTML5 history 模式还是 hash 模式，router-link 的表现行为一致，所以，当要切换路由模式，或者在 IE9 降级使用 hash 模式，无须作任何变动。
在 HTML5 history 模式下，router-link 会守卫点击事件，让浏览器不再重新加载页面。
在 HTML5 history 模式下，使用 router-link 的 base 选项之后，所有的 to 属性都不需要写 (基路径) 了。

<router-link> Props

  • to
    类型: string | Location
    设置目标路由的链接。
    关联: #router.push# #router.replace#

  • active-class
    类型: string
    默认值: "router-link-active"
    设置当前路径包含 to 属性值时使用的 CSS 类名。
    关联: #linkActiveClass#

  • exact-active-class
    类型: string
    默认值: "router-link-exact-active"
    设置当前路径完全匹配 to 属性值时使用的 CSS 类名。
    关联: #linkExactActiveClass#

  • tag
    类型: string
    默认值: "a"
    指定要 <router-link> 渲染成何种标签。
    注意: 不论是何种标签都会监听点击，触发导航。

<router-view>

<router-view> 组件用于渲染路径匹配到的视图组件。

非 router-view 使用的属性，都直接传给渲染的组件。router-view 渲染的组件还可以内嵌自己的 router-view，根据嵌套路径，渲染嵌套组件。

<router-view> Props

  • name
    类型: string
    默认值: "default"
    用于命名视图渲染对应的路由配置中 components 下的相应组件。
