Vue.nextTick

Vue.nextTick( [callback, context] )

参数：
{Function} [callback]
{Object} [context]
返回值：Promise - 在没有提供回调且在支持 Promise 的环境中。
作用：配置 DOM 更新后的回调操作。
关联：vm.$nextTick
关键字：异步更新 #延迟回调#

vm.$nextTick 和 Vue.nextTick 的区别？

Vue.nextTick 内部函数的 this 指向 window。
vm.$nextTick 内部函数的 this 指向 Vue 实例对象。

选项/数据

data

类型：Object | Function
限制：组件的定义只接受 function。
作用：配置 Vue 实例要用到的数据对象。
关联：vm.$data.([property]) vm.[property] created
关键字：#数据对象# 响应式

methods

类型：{ [key: string]: Function }
限制：如果使用箭头函数 this 将不会指向 Vue 实例，不过可以将其实例作为函数的第一个参数来访问。
作用：配置 Vue 实例要用到的方法函数。
关联：vm.[key]() computed created
关键字：事件处理

computed

类型：{ [key: string]: Function | { get: Function, set: Function } }
限制：如果使用箭头函数 this 将不会指向 Vue 实例，不过可以将其实例作为函数的第一个参数来访问。
作用：配置 Vue 实例要用到的计算属性。
关联：vm.[key] methods watch {{ Mustache }}
关键字：#计算属性# 响应式

{{ Mustache }} 和 computed 的区别？

{{ Mustache }} 在模板中放入太多的逻辑运算会让模板过重难以维护和阅读，如果重复的使用还会影响到性能。
computed 计算属性只在相关响应式依赖发生改变时它们才会重新求值，并更新所有绑定的模板，提高性能的同时也减少了维护成本，另外计算属性还提供了设置依赖的 setter 方法。

methods 和 computed 的区别？

methods 调用方法不会缓存，不论是否存在依赖关系，重新渲染时总会再次执行函数，从而会影响到性能。
computed 计算属性是基于它们的响应式依赖进行缓存的，如果依赖没有发生改变，则返回之前缓存的计算结果，而不必再次执行函数，从而效率更高。对于非响应式依赖，则计算属性不会被更新。

watch

类型：{ [key: string]: string | Function | Object | Array }
限制：如果使用箭头函数 this 将不会指向 Vue 实例，不过可以将其实例作为函数的第一个参数来访问。
作用：配置 Vue 实例要侦听的数据变化时要触发的回调函数。
关联：vm.$watch() computed
关键字：#侦听器# #侦听属性#

watch 和 computed 的区别？

watch 当监听的数据变化时，执行回调函数，用于异步或开销较大的操作。
computed 当依赖的数据变化时，处理简单的操作并重新渲染页面，用于多数据影响单数据的响应式操作。

props

类型：Array<string> | Object
选项：
type：String | Number | Boolean | Array | Object | Date | Function | Symbol | 任何自定义构造函数 | 上述内容组成的数组
检查一个 prop 是否是给定的类型，否则抛出警告。
default：any
为该 prop 指定一个默认值。对象或数组的默认值必须从一个工厂函数返回。
required：Boolean
定义该 prop 是否是必填项。在非生产环境中，如果这个值为 truthy 且该 prop 没有被传入的，则一个控制台警告将会被抛出。
validator：Function
自定义验证函数会将该 prop 的值作为唯一的参数代入。在非生产环境下，如果该函数返回一个 falsy 的值 (也就是验证失败)，一个控制台警告将会被抛出。
限制：使用对象允许配置高级选项，如类型检测、自定义验证和设置默认值。
作用：用于接收来自父组件的数据。
关联：$parent provide/inject
关键字：组件

选项/DOM

el

类型：string | Element
限制：只在用 new 创建实例时生效。
作用：配置 Vue 实例要控制的区域（元素）。
关联：vm.$el vm.$mount()
关键字：#挂载目标# #挂载元素#

template

类型：string
限制：如果 Vue 选项中包含渲染函数，该模板将被忽略。
作用：一个字符串模板作为 Vue 实例的标识使用，模板将会替换挂载的元素。
关键字：生命周期 #分发插槽# #模板定义#

render

类型：(createElement: () => VNode) => VNode
限制：Vue 选项中的 render 函数若存在，则 Vue 构造函数不会从 template 选项或通过 el 选项指定的挂载元素中提取出的 HTML 模板编译渲染函数。
作用：利用 JavaScript 创建虚拟节点来替代字符串模板的渲染方案。
关联：createElement
关键字：渲染函数 JSX

选项/生命周期钩子

beforeCreate

类型：Function
限制：如果使用箭头函数 this 将不会指向 Vue 实例，不过可以将其实例作为函数的第一个参数来访问。
作用：在实例初始化之后，数据观测 (data observer) 和 event/watcher 事件配置之前被调用。
关键字：生命周期

created

类型：Function
限制：如果使用箭头函数 this 将不会指向 Vue 实例，不过可以将其实例作为函数的第一个参数来访问。
作用：在实例创建完成后被立即调用。在这一步，实例已完成以下的配置：数据观测 (data observer)，property 和方法的运算，watch/event 事件回调。
关联：data methods
关键字：生命周期 #数据请求#

beforeMount

类型：Function
限制：如果使用箭头函数 this 将不会指向 Vue 实例，不过可以将其实例作为函数的第一个参数来访问。该钩子在服务器端渲染期间不被调用。
作用：在挂载开始之前被调用，此时模板已经编译完成，只是未将生成的模板替换 el 对应的元素。
关键字：生命周期

mounted

类型：Function
限制：如果使用箭头函数 this 将不会指向 Vue 实例，不过可以将其实例作为函数的第一个参数来访问。该钩子在服务器端渲染期间不被调用。注意 mounted 不会保证所有的子组件也都一起被挂载。
作用：实例被挂载后调用，这时 el 被新创建的 vm.$el 替换，并且随实例挂载到了一个文档内的元素上。
关联：vm.$nextTick
关键字：生命周期 #DOM操作#

beforeUpdate

类型：Function
限制：如果使用箭头函数 this 将不会指向 Vue 实例，不过可以将其实例作为函数的第一个参数来访问。该钩子在服务器端渲染期间不被调用。
作用：数据更新时调用，发生在虚拟 DOM 打补丁之前。
关键字：生命周期

updated

类型：Function
限制：如果使用箭头函数 this 将不会指向 Vue 实例，不过可以将其实例作为函数的第一个参数来访问。该钩子在服务器端渲染期间不被调用。注意 updated 不会保证所有的子组件也都一起被重绘。
作用：在因数据更改导致的虚拟 DOM 重新渲染和打补丁之后调用，此时组件 DOM 已经更新。
关联：vm.$nextTick
关键字：生命周期 #DOM操作#

beforeDestroy

类型：Function
限制：如果使用箭头函数 this 将不会指向 Vue 实例，不过可以将其实例作为函数的第一个参数来访问。该钩子在服务器端渲染期间不被调用。
作用：实例销毁之前调用，此时实例仍然完全可用。
关键字：生命周期

destroyed

类型：Function
限制：如果使用箭头函数 this 将不会指向 Vue 实例，不过可以将其实例作为函数的第一个参数来访问。该钩子在服务器端渲染期间不被调用。
作用：实例销毁后调用。该钩子被调用后，对应 Vue 实例的所有指令都被解绑，所有的事件监听器被移除，所有的子实例也都被销毁。
关键字：生命周期

activated

类型：Function
限制：如果使用箭头函数 this 将不会指向 Vue 实例，不过可以将其实例作为函数的第一个参数来访问。该钩子在服务器端渲染期间不被调用。
作用：被 keep-alive 缓存的组件激活时调用。
关联：keep-alive
关键字：动态组件

deactivated

类型：Function
限制：如果使用箭头函数 this 将不会指向 Vue 实例，不过可以将其实例作为函数的第一个参数来访问。该钩子在服务器端渲染期间不被调用。
作用：被 keep-alive 缓存的组件停用时调用。
关联：keep-alive
关键字：动态组件

选项/组合

provide/inject

类型：
provide：Object | () => Object
inject：Array<string> | { [key: string]: string | Symbol | Object }
限制：provide 和 inject 绑定并不是可响应的。
作用：用于祖先组件向其所有子孙后代注入一个依赖，不论组件层次有多深，并在其上下游关系成立的时间里始终生效。
关联：props
关键字：依赖注入

provide/inject 和 props 的区别？

使用 provide/inject 时祖先组件无法知晓哪些后代组件使用它提供的 property。
使用 provide/inject 时后代组件无法知晓被注入的 property 来自哪里。

选项/其它

inheritAttrs

类型：boolean
限制：此选项不影响 class 和 style 绑定。
作用：用于取消以下默认行为：对于组件中未被 prop 注册的父作用域 attribute，将作为普通的 HTML attribute 应用在组件的根元素上，默认值为 true 即采用默认行为。
关联：$attrs $listeners
关键字：组件

model

类型：{ prop?: string, event?: string }
限制：默认情况下，一个组件上的 v-model 会把 value 用作 prop 且把 input 用作 event。
作用：允许一个自定义组件在使用 v-model 时定制 prop 和 event。
关联：v-model
关键字：组件

name

类型：string
限制：只有作为组件选项时起作用，并且如用 Vue.component() 注册组件时，全局 ID 自动作为组件的 name。
作用：使组件语义化，可允许组件模板递归地调用自身，也便于调试。
关联：v-model
关键字：组件

functional

类型：boolean
限制：仅用 render 函数返回虚拟节点，可降低渲染开销。
作用：使组件无状态 (没有 data) 无生命周期和无实例 (没有 this 上下文)。
关联：context
关键字：函数式组件

指令

v-text

类型：string
限制：只能替换元素中所有的文本。
作用：更新元素的 textContent。[1]
关联：{{ Mustache }}
关键字：数据绑定

v-text 和 {{ Mustache }} 的区别？

v-text 优先级高于 {{ Mustache }}。
v-text 替换元素中所有的文本，而 {{ Mustache }} 只替换自己，不清空元素内容。

v-html

类型：string
限制：内容按普通 HTML 插入，不会作为 Vue 模板进行编译。
作用：更新元素的 innerHTML。
关键字：数据绑定 #XSS攻击#

v-pre

限制：Mustache 标签以原始方式显示，不会作为 Vue 模板进行编译。
作用：跳过这个元素和它的子元素的编译过程，加快编译速度。
关联：{{ Mustache }}

v-cloak

限制：在编译结束前，配合 CSS 规则，可以隐藏未编译的 Mustache 标签。
作用：这个指令保持在元素上直到关联实例结束编译。
关联：{{ Mustache }}

v-once

限制：随后的重新渲染，元素/组件及其所有的子节点将被视为静态内容并跳过。
作用：只渲染元素和组件一次，以优化更新性能。
关键字：数据绑定 控制更新

v-if

类型：any
限制：配合 <template> 元素将提出它的内容作为条件块实现多个元素的切换。
作用：根据表达式的值的 truthiness 来有条件地渲染元素，当条件变化时该指令触发过渡效果。
关联：<template> v-else v-else-if v-show key
关键字：#条件渲染#

v-else

限制：前一兄弟元素必须有 v-if 或 v-else-if。
作用：为 v-if 或者 v-else-if 添加“else 块”。
关联：<template> v-if v-else-if v-show key
关键字：#条件渲染#

v-else-if

类型：any
限制：前一兄弟元素必须有 v-if 或 v-else-if。
作用：表示 v-if 的“else if 块”。可以链式调用。
关联：<template> v-if v-else v-show key
关键字：#条件渲染#

v-show

类型：any
限制：不能配合 <template> 元素，不支持 v-else / v-else-if。
作用：根据表达式之真假值切换元素的 display CSS 属性，当条件变化时该指令触发过渡效果。
关联：v-if v-else v-else-if
关键字：#条件渲染#

v-if 和 v-show 的区别？

v-if 是惰性的，如果在初始渲染时条件为假，则什么也不做，直到条件第一次变为真时，才会开始渲染条件块。v-show 则不管初始条件是什么，元素总是会被渲染，并且只是简单地基于 CSS 进行切换。
v-if 有更高的切换开销，v-show 有更高的初始渲染开销，如果需要非常频繁地切换，则使用 v-show 较好，如果在运行时条件很少改变，则使用 v-if 较好。

v-bind

缩写：:
类型：any (with argument) | Object (without argument)
参数：attrOrProp (optional)
限制：参考 数据绑定
修饰符：
.prop - 作为一个 DOM property 绑定而不是作为 attribute 绑定。
.camel - (2.1.0+) 将 kebab-case attribute 名转换为 camelCase。
.sync - (2.3.0+) 语法糖，会扩展成一个更新父组件绑定值的 v-on 侦听器。
作用：动态地绑定一个或多个 attribute，或一个组件 prop 到表达式。
关键字：#动态绑定# 数据绑定

v-bind.sync 和 v-model 的区别？

两者都是用于实现双向数据传递的，实现方式都是语法糖，最终通过 v-bind 和 v-on 来达成目的。
vue 1.x 的 .sync 可以完全在子组件中修改父组件的状态，vue 2.3 之后跟 v-model 的实现原理是一样的，使用场景略微有区别。
当一个组件对外只暴露一个受控的状态，且都符合统一标准的时候，可以使用 v-model 处理。.sync 则更为灵活，凡是需要双向数据传递时，都可以去使用。

v-on

缩写：@
类型：Function | Inline Statement | Object
参数：event
限制：参考 事件处理
修饰符：
.stop - 调用 event.stopPropagation() 阻止冒泡。
.prevent - 调用 event.preventDefault() 阻止默认行为。
.capture - 添加事件侦听器时使用事件捕获模式。
.self - 只当事件是从侦听器绑定的元素本身触发时才触发回调。
.{keyCode | keyAlias} - 只当事件是从特定键触发时才触发回调。
.native - 监听组件根元素的原生事件。
.once - 只触发一次回调 可用于组件。
.left - (2.2.0) 只当点击鼠标左键时触发。
.right - (2.2.0) 只当点击鼠标右键时触发。
.middle - (2.2.0) 只当点击鼠标中键时触发。
.passive - (2.3.0) 不阻止事件的默认行为 可用于提升移动端的性能。
.ctrl - (2.1.0) 仅在按下 ctrl 按键时才触发。
.alt - (2.1.0) 仅在按下 alt 按键时才触发。
.shift - (2.1.0) 仅在按下 shift 按键时才触发。
.meta - (2.1.0) 仅在按下 meta 按键时才触发。
.exact - (2.5.0) 配合系统修饰符控制精确触发。
作用：绑定事件监听器，事件类型由参数指定。
关键字：事件处理 #事件监听#

v-for

类型：Array | Object | number | string | Iterable (2.6 新增)
限制：当和 v-if 一起使用时，v-for 的优先级更高，不推荐同时使用 v-if 和 v-for。
作用：基于源数据多次渲染元素或模板块。
关联：<template> key
关键字：列表渲染

v-model

类型：随表单控件类型不同而不同。
限制：用在组件或表单 <input>、<textarea> 及 <select> 元素上。
修饰符：
.lazy - 监听 change 事件而不是默认的 input 事件。
.number - 将输入字符串转为有效的数值类型，如无法完成转换则返回原始的值。
.trim - 过滤输入的首尾空格。
作用：在表单控件或者组件上创建双向绑定，并会根据控件类型自动选取正确的方法来更新元素。
关联：v-bind v-on
关键字：#双向数据绑定# 表单控件绑定

v-slot

缩写：#
类型：可选，可放置在函数参数位置的 JavaScript 表达式。
参数：插槽名 (可选，默认值是 default)
限制：用于 <template> 上或单独带 prop 的默认插槽组件上。
作用：提供具名插槽或需要接收 prop 的插槽。
关联：slot $slots
关键字：插槽

  1. textContent 和 innerText
    设置文本替换时，两者都会把指定节点下的所有子节点也一并替换掉。
    textContent 会获取所有元素的文本内容，包括 <script> 和 <style> 元素，然而 innerText 不会。
    textContent 不受 CSS 样式的影响，会返回隐藏元素的文本，然而 innerText 不会，它受 CSS 样式的影响，会触发重排，从而影响性能。
    textContent 作为标准方法，只支持 IE8+ 以上的浏览器，然而 innerText 是 IE 引入的，对 IE 支持更友好。在最新的浏览器中，两个都可以使用。 ↩︎
