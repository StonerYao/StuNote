Vuex

Vuex 是一个专为 Vue 应用程序开发的状态管理模式，用于实现多组件的状态共享。多用于中大型单页应用。让代码变得更结构化且易维护。

Vuex 采用集中式存储管理应用的所有组件的状态，任何组件都能获取状态或者触发行为，并以相应的规则保证状态以一种可预测的方式发生变化。

Vuex

安装

  • 直接下载 / CDN: https://unpkg.com/vuex
  • NPM: npm install vuex --save

注意: Vuex 依赖 Promise。如果所支持的浏览器没有实现 Promise (比如 IE)，需要先安装一个 polyfill 的库，例如 es6-promise。

基本使用

  1. 引入 Vuex

    <!-- 直接下载 / CDN -->
<script src="/path/to/vue.js"></script>
<script src="/path/to/vuex.js"></script>

    // NPM + 模块化
import Vue from 'vue'
import Vuex from 'vuex'

Vue.use(Vuex)

  2. 创建 store 实例

    const store = new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment (state) {
      state.count++
    }
  }
})

  3. 创建和挂载根实例

    const app = new Vue({
  // 等同于 `store: store`
  store
}).$mount('#app')

  4. 在组件中使用状态

    // 获取状态
computed: {
  count () {
    return this.$store.state.count
  }
},
// 修改状态
methods: {
  increment () {
    this.$store.commit('increment')
  }
}

Store

Store (仓库) 由 Vuex.Store 方法创建，作为 Vuex 应用的核心容器，每个应用中只能包含一个仓库实例。

可以在根实例中注册 store 选项，将仓库实例注入到其下的所有子组件中，且子组件能通过 this.$store 访问到。

State

State (状态) 用于存储全部的应用层级状态，作为唯一的数据源。Vuex 的状态存储是响应式的，且只能通过显式地提交 mutation 来改变。

可以在子组件中通过计算属性从仓库实例中获取状态。

注意: 存储在 Vuex 中的数据和 Vue 实例中的 data 遵循相同的规则。

可以使用 mapState 辅助函数将 store 中的 state 映射到局部计算属性。

// 在单独构建的版本中辅助函数为 Vuex.mapState
import { mapState } from 'vuex'

export default {
  // ...
  computed: mapState({
    count: state => state.count,
    
    // 等同于 state => state.count
    countAlias: 'count',
    
    // 使用 this 获取局部状态
    countPlusLocalState (state) {
      return state.count + this.localCount
    }
  })
}

当映射的计算属性的名称与 state 的子节点名称相同时，也可以给 mapState 传一个字符串数组。

computed: mapState([
  // 映射 this.count 为 store.state.count
  'count'
])

使用对象展开运算符将 mapState 创建的计算属性与局部计算属性混合使用。

computed: {
  localComputed () { /* ... */ },
  // 将此对象混入到外部对象中
  ...mapState({
    // ...
  })
}

Getter

Getter 用于从 store 中的 state 中派生出一些状态，可以认为是 store 的计算属性。

const store = new Vuex.Store({
  state: {
    todos: [
      { id: 1, text: '...', done: true },
      { id: 2, text: '...', done: false }
    ]
  },
  getters: {
    doneTodos: state => {
      return state.todos.filter(todo => todo.done)
    }
  }
})

Getter 接受 state 作为其第一个参数，接受其它 getters 作为第二个参数。

getters: {
  // ...
  doneTodosCount: (state, getters) => {
    return getters.doneTodos.length
  }
}

可以在任何组件中通过属性访问它，此时 getter 的返回值会根据它的依赖被缓存起来，且只有当它的依赖值发生了改变才会被重新计算。

computed: {
  doneTodosCount () {
    return this.$store.getters.doneTodosCount // -> 1
  }
}

当 getter 返回一个函数时，可以给 getter 传参并通过调用方法访问它，此时不会缓存结果。

getters: {
  // ...
  getTodoById: (state) => (id) => {
    return state.todos.find(todo => todo.id === id)
  }
}

this.$store.getters.getTodoById(2) // -> { id: 2, text: '...', done: false }

可以使用 mapGetters 辅助函数将 store 中的 getters 映射到局部计算属性。

import { mapGetters } from 'vuex'

export default {
  // ...
  computed: {
  // 使用对象展开运算符将 getter 混入 computed 对象中
    ...mapGetters([
      'doneTodosCount',
      'anotherGetter',
      // ...
    ])
  }
}

使用对象形式，可以给 getter 属性另取一个名字。

...mapGetters({
  // 把 this.doneCount 映射为 this.$store.getters.doneTodosCount
  doneCount: 'doneTodosCount'
})

Mutation

Mutation 作为唯一的方法，用于更改 Vuex store 中的状态，类似于事件。

const store = new Vuex.Store({
  state: {
    count: 1
  },
  mutations: {
    increment (state) {
      // 变更状态
      state.count++
    }
  }
})

Mutation 接受 state 作为其第一个参数，接受可选的提交载荷 (Payload) 作为第二个参数。

mutations: {
  // ...
  next (state) {
    state.count++
  }
  increment (state, payload) {
    state.count += payload
  },
  reduction (state, payload) {
    state.count -= payload.amount
  }
}

可以在任何组件中通过调用 this.$store.commit 方法提交一个 mutation。另外在提交载荷时，需要传入额外的参数，当载荷包含多个字段时，可以采用对象的形式传入。

this.$store.commit('next')
this.$store.commit('increment', 10)
this.$store.commit('reduction', { amount: 10 })

提交 mutation 的另一种方式是直接使用包含 type 属性的对象，对象将作为载荷传给 mutation 函数。

this.$store.commit({
  type: 'reduction',
  amount: 10
})

注意: Mutation 需遵守 Vue 的响应规则，因此最好提前在 store 中初始化好所有所需属性。当需要在对象上添加新属性时，应该使用 Vue.set() 方法, 或者利用对象展开运算符以新对象替换老对象。

在大型项目中可以使用常量替代 Mutation 事件类型，以使整个 app 包含的 mutation 一目了然。

// mutation-types.js
export const SOME_MUTATION = 'SOME_MUTATION'

// store.js
import Vuex from 'vuex'
import { SOME_MUTATION } from './mutation-types'

const store = new Vuex.Store({
  state: { ... },
  mutations: {
    // 使用 ES2015 风格的计算属性命名功能 用一个常量作为函数名
    [SOME_MUTATION] (state) {
      // mutate state
    }
  }
})

Mutation 作为更改 Vuex store 中状态的唯一的方法，所以它的回调必须是同步函数，以方便追踪状态的改变，同时也方便使用 devtools 进行调试。并且在 Vuex store 的严格模式下 (创建 store 时传入 strict: true)，任何 mutation 处理函数以外修改 state 都会抛出错误。

// 确保在生产环境中关闭严格模式
const store = new Vuex.Store({
  // ...
  strict: process.env.NODE_ENV !== 'production'
})

注意: 不要在生产环境下启用严格模式。严格模式会深度监测状态树来检测不合规的状态变更，从而造成性能损失。

当 Vuex 与表单双向绑定联用时，可以通过 v-bind 和 v-on 模拟 v-model，也可以通过设置计算属性的 setter，来实现表单中数据与 Vuex store 中状态的双向绑定功能。

<input v-model="msg" />

computed: {
  msg: {
    get () {
      return this.$store.state.obj.msg;
    },
    set (value) {
      this.$store.commit('update', { value });
    }
  }
}

可以使用 mapMutations 辅助函数将 store 中的 mutations 的 commit 调用映射到局部方法。

import { mapMutations } from 'vuex'

export default {
  // ...
  methods: {
    ...mapMutations([
      // 将 this.increment() 映射为 this.$store.commit('increment')
      'increment',

      // 将 this.incrementBy(amount) 映射为 this.$store.commit('incrementBy', amount)
      'incrementBy'
    ]),
    ...mapMutations({
      // 将 this.add() 映射为 this.$store.commit('increment')
      add: 'increment'
    })
  }
}

Action

Action 通过提交 mutation 从而更改 Vuex store 中的状态。

Action 函数接受一个与 store 实例具有相同方法和属性的 context 对象作为其第一个参数，因此可以调用 context.commit 提交一个 mutation，或者通过 context.state 和 context.getters 来获取 state 和 getters。也可以将 context 对象的方法和属性通过 ES2015 的参数解构来简化代码。

const store = new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment (state) {
      state.count++
    }
  },
  actions: {
    increment (context) {
      context.commit('increment')
    }
  }
})

注意: context 对象并不是 store 实例本身。

同样 Action 函数也接受可选的提交载荷作为第二个参数。

actions: {
  // ...
  increment (context, payload) {
    context.commit('increment', payload)
  }
}

可以在任何组件中通过调用 this.$store.dispatch 方法分发 Action。另外 Actions 也支持同样的载荷方式和对象方式进行分发。

this.$store.dispatch('increment')

// 以载荷形式分发
this.$store.dispatch('increment', { amount: 10 })

// 以对象形式分发
this.$store.dispatch({
  type: 'increment',
  amount: 10
})

与 mutation 不同，Action 可以包含任意异步操作。

actions: {
  incrementAsync ({ commit }) {
    setTimeout(() => {
      commit('increment')
    }, 1000)
  }
}

store.dispatch 方法可以处理被触发的 action 的处理函数返回的 Promise，并且 store.dispatch 仍旧返回 Promise。从而可以组合多个 action，以处理更加复杂的异步流程。

actions: {
  actionA ({ commit }) {
    return new Promise((resolve, reject) => {
      setTimeout(() => {
        commit('someMutation')
        resolve()
      }, 1000)
    })
  },
  actionB ({ dispatch, commit }) {
    return dispatch('actionA').then(() => {
      commit('someOtherMutation')
    })
  }
}

利用 async / await 组合多个 action。

// 假设 getData() 和 getOtherData() 返回的是 Promise

actions: {
  async actionA ({ commit }) {
    commit('gotData', await getData())
  },
  async actionB ({ dispatch, commit }) {
    await dispatch('actionA') // 等待 actionA 完成
    commit('gotOtherData', await getOtherData())
  }
}

注意: 一个 store.dispatch 在不同模块中可以触发多个 action 函数。在这种情况下，只有当所有触发函数完成后，返回的 Promise 才会执行。

可以使用 mapActions 辅助函数将 store 中的 actions 的 dispatch 调用映射到局部方法。

import { mapActions } from 'vuex'

export default {
  // ...
  methods: {
    ...mapActions([
      // 将 this.increment() 映射为 this.$store.dispatch('increment')
      'increment',

      // 将 this.incrementBy(amount) 映射为 this.$store.dispatch('incrementBy', amount)
      'incrementBy'
    ]),
    ...mapActions({
      // 将 this.add() 映射为 this.$store.dispatch('increment')
      add: 'increment'
    })
  }
}

Module

Module 用于将 store 分割成模块，每个模块拥有自己的 state、mutation、action、getter、甚至是嵌套子模块。

const moduleA = {
  state: () => ({ ... }),
  mutations: { ... },
  actions: { ... },
  getters: { ... }
}

const moduleB = {
  state: () => ({ ... }),
  mutations: { ... },
  actions: { ... }
}

const store = new Vuex.Store({
  modules: {
    a: moduleA,
    b: moduleB
  }
})

store.state.a // -> moduleA 的状态
store.state.b // -> moduleB 的状态

注意: 在模块中使用函数来声明局部状态，可以避免在模块重用时数据污染的问题。

模块的局部状态，在模块内部的 mutation 和 getter 中通过接收的第一个参数 state 暴露出来，在模块内部的 action 中通过 context.state 暴露出来。

根节点的全局状态，在模块内部的 getter 中通过接收的第三个参数 rootState 暴露出来，在模块内部的 action 中通过 context.rootState 暴露出来。

const moduleA = {
  state: () => ({
    count: 0
  }),
  mutations: {
    increment (state) {
      state.count++
    }
  },
  getters: {
    sumWithRootCount (state, getters, rootState) {
      return state.count + rootState.count
    }
  },
  actions: {
    incrementIfOddOnRootSum ({ state, commit, rootState }) {
      if ((state.count + rootState.count) % 2 === 1) {
        commit('increment')
      }
    }
  }
}

模块内部的 action、mutation 和 getter 默认注册在全局的命名空间中，从而使得多个模块能够对同一 mutation 或 action 作出响应。

如果希望模块具有更高的封装度和复用性，可以通过添加 namespaced: true 的方式使其成为带命名空间的模块。并且当模块被注册后，它的所有 getter、action 及 mutation 都会自动根据模块注册的路径调整命名。对于嵌套模块，将继承父模块的命名空间。

const store = new Vuex.Store({
  modules: {
    account: {
      namespaced: true,

      // 模块内容
      state: () => ({
        foo: 'bar' // -> state.account.foo
      }),
      getters: {
        isAdmin () { ... } // -> getters['account/isAdmin']
      },
      actions: {
        login () { ... } // -> dispatch('account/login')
      },
      mutations: {
        login () { ... } // -> commit('account/login')
      },

      // 嵌套模块
      modules: {
        myPage: {
          state: () => ({ ... }),
          getters: {
            profile () { ... } // -> getters['account/profile']
          }
        },

        // 嵌套命名空间
        posts: {
          namespaced: true,

          state: () => ({ ... }),
          getters: {
            popular () { ... } // -> getters['account/posts/popular']
          }
        }
      }
    }
  }
})

在带命名空间的模块内，可以通过传入 getter 中的第二个参数 getters 以及传入 action 中的 context.getters 访问局部 getter。也可以通过传入 getter 中的第四个参数 rootGetters 以及传入 action 中的 context.rootGetters 访问全局 getter。

另外在带命名空间的模块内，dispatch 和 commit 只能在局部命名空间内分发 action 或提交 mutation。可以将 { root: true } 作为第三参数传给 dispatch 或 commit，从而在全局命名空间内分发 action 或提交 mutation。

modules: {
  foo: {
    namespaced: true,

    getters: {
      someGetter (state, getters, rootState, rootGetters) {
        getters.someOtherGetter // -> 'foo/someOtherGetter'
        rootGetters.someOtherGetter // -> 'someOtherGetter'
      },
      someOtherGetter: state => { ... }
    },

    actions: {
      someAction ({ dispatch, commit, getters, rootGetters }) {
        getters.someGetter // -> 'foo/someGetter'
        rootGetters.someGetter // -> 'someGetter'

        dispatch('someOtherAction') // -> 'foo/someOtherAction'
        dispatch('someOtherAction', null, { root: true }) // -> 'someOtherAction'

        commit('someMutation') // -> 'foo/someMutation'
        commit('someMutation', null, { root: true }) // -> 'someMutation'
      },
      someOtherAction (ctx, payload) { ... }
    }
  }
}

在带命名空间的模块内，可以通过对象形式注册 action，添加 root: true，并将这个 action 的定义放在函数 handler 中，注册全局 action。

{
  actions: {
    someOtherAction ({dispatch}) {
      dispatch('someAction')
    }
  },
  modules: {
    foo: {
      namespaced: true,

      actions: {
        someAction: {
          root: true,
          handler (namespacedContext, payload) { ... } // -> 'someAction'
        }
      }
    }
  }
}

当使用 mapState、mapGetters、mapActions 和 mapMutations 这些函数来绑定带命名空间的模块时，可以将模块的空间名称字符串作为第一个参数传递给上述函数，这样所有绑定都会自动将该模块作为上下文。

computed: {
  ...mapState('some/nested/module', {
    a: state => state.a,
    b: state => state.b
  })
},
methods: {
  ...mapActions('some/nested/module', [
    'foo', // -> this.foo()
    'bar' // -> this.bar()
  ])
}

也可以通过使用 createNamespacedHelpers 创建基于某个命名空间辅助函数，它返回一个对象，对象里有新的绑定在给定命名空间值上的组件绑定辅助函数。

import { createNamespacedHelpers } from 'vuex'

const { mapState, mapActions } = createNamespacedHelpers('some/nested/module')

export default {
  computed: {
    // 在 `some/nested/module` 中查找
    ...mapState({
      a: state => state.a,
      b: state => state.b
    })
  },
  methods: {
    // 在 `some/nested/module` 中查找
    ...mapActions([
      'foo',
      'bar'
    ])
  }
}

为了使得其他 Vue 插件可以通过在 store 中附加新模块的方式来使用 Vuex 管理状态，可以在 store 创建之后使用 store.registerModule 方法动态注册模块。也可以使用 store.unregisterModule(moduleName) 来动态卸载模块，但不能使用此方法卸载静态模块 (即创建 store 时声明的模块)。另外，可以通过 store.hasModule(moduleName) 方法检查该模块是否已经被注册到 store。

import Vuex from 'vuex'

const store = new Vuex.Store({ /* 选项 */ })

// 注册模块 myModule
store.registerModule('myModule', {
  // ...
})
// 注册嵌套模块 nested/myModule
store.registerModule(['nested', 'myModule'], {
  // ...
})

store.registerModule(moduleName, module, preserveState) 方法接收 preserveState 选项作为第三个参数，当设置 preserveState: true 时，该模块会被注册，action、mutation 和 getter 会被添加到 store 中，但是 state 不会。多用于 store 的 state 已经包含了这个 module 的 state，并且不希望将其覆写的情况。

Vuex.Store 构造器选项

state (构造器选项)

类型: Object | Function
用于定义 Vuex store 中的状态。

getters (构造器选项)

类型: { [key: string]: Function }
用于定义 Vuex store 中派生出的状态。

mutations

类型: { [type: string]: Function }
用于定义更改 Vuex store 中状态的方法。

actions

类型: { [type: string]: Function }
用于定义提交 Vuex store 中 mutation 的方法。

modules

类型: { [type: string]: Object }
用于定义 Vuex store 中的模块。

strict

类型: boolean
默认值: false
用于使 Vuex store 进入严格模式。

Vuex.Store 实例属性

state (实例属性)

类型: Object
根状态，只读。

getters (实例属性)

类型: Object
注册的 getter，只读。

Vuex.Store 实例方法

commit

类型:
commit(type: string, payload?: any, options?: Object)
commit(mutation: Object, options?: Object)
用于提交 mutation。

dispatch

类型:
dispatch(type: string, payload?: any, options?: Object): Promise<any>
dispatch(action: Object, options?: Object): Promise<any>
用于分发 action。

registerModule

类型:
registerModule(path: string | Array<string>, module: Module, options?: Object)
用于注册动态模块。

unregisterModule

类型:
unregisterModule(path: string | Array<string>)
用于卸载动态模块。

hasModule

类型:
hasModule(path: string | Array<string>): boolean
用于检查该模块的名字是否已经被注册。

组件绑定的辅助函数

mapState

类型: mapState(namespace?: string, map: Array<string> | Object<string | function>): Object
返回值: Object
为组件创建计算属性以返回 Vuex store 中的 state。

mapGetters

类型: mapGetters(namespace?: string, map: Array<string> | Object<string>): Object
返回值: Object
为组件创建计算属性以返回 Vuex store 中的 getter。

mapMutations

类型: mapMutations(namespace?: string, map: Array<string> | Object<string | function>): Object
返回值: Object
为组件创建方法以提交 Vuex store 中的 mutation。

mapActions

类型: mapActions(namespace?: string, map: Array<string> | Object<string | function>): Object
返回值: Object
为组件创建方法以分发 Vuex store 中的 action。

createNamespacedHelpers

类型: createNamespacedHelpers(namespace: string): Object
返回值: Object
为组件创建基于命名空间的组件绑定辅助函数。
