Build Performance 构建性能

减少模块解析(缩短构建时间)

在 webpack 打包流程中，模块解析包括：抽象语法树分析、依赖分析、模块语法替换。

模块解析

如果某个模块不做解析，该模块经过 loader 处理后的代码就是最终代码。如果没有 loader 对该模块进行处理，该模块的源码就是最终打包结果的代码。

减少模块解析

对于无其他依赖的模块不需要解析，比如一些已经打包好的第三方库，例如 jquery。

对于不需要解析的模块，可以在 webpack 配置文件中配置 module.noParse 选项，它的值是一个正则，被正则匹配到的模块不会解析。

优化 loader 处理(缩短构建时间)

限制 loader 的应用范围

对于某些库，不使用 loader。例如 babel-loader 可以转换 ES6 或更高版本的语法，可是有些库例如 lodash 本身就是用 ES5 语法书写的不需要转换，使用 babel-loader 反而会浪费构建时间。

通过在 webpack 配置文件中设置 module.rule.exclude 或 module.rule.include 的值，排除或仅包含需要应用 loader 的场景。

缓存 loader 的结果

如果某个文件内容不变，经过相同的 loader 解析后，解析后的结果也不变。可以将 loader 的解析结果保存下来，让后续的解析直接使用保存的结果。

通过在 webpack 中使用 cache-loader 加载器，缓存 loader 的结果。

webpack 在处理 loaders 的过程中还包含分析 loader 中 pitch 的过程，如果 pitch 有返回值则将源代码返回给前一个 loader 处理，否则继续按照流程处理 loaders。

分析 pitch

cache-loader 利用这一原理，判断后续 loaders 是否存在缓存，如有缓存则使用保存的缓存结果。

为 loader 的运行开启多线程

把 loader 放到线程池的其他线程中运行，以提高构建效率。

通过在 webpack 中使用 thread-loader 加载器，开启一个线程池，线程池中包含适量的线程，让后续的加载器在其他线程中运行。

由于后续的 loader 会放到新的线程中，所以后续的 loader 不能做以下事情:

  • 使用 webpack api 生成文件。
  • 使用自定义的 plugin api。
  • 访问 webpack options。

特别注意，开启和管理线程需要消耗时间，在小型项目中使用 thread-loader 反而会增加构建时间。

HMR 热替换(缩短代码改动到效果呈现的时间)

当使用 webpack-dev-server 时，如果代码改动，浏览器默认会重新请求所有资源，并刷新页面重新加载所有内容。

没有使用热替换时的流程

而使用了热替换后，流程发生了变化，浏览器仅请求改动的资源，并重新加载改动的内容，不会刷新页面。

使用热替换时的流程

通过在 webpack 配置文件中设置 devServer.hot 的值为 true 开启热替换，并且需要在入口文件中加入如下代码。

// 这是入口文件 会参与最终运行

if(module.hot){ // 是否开启了热替换
    module.hot.accept() // 接受热替换
}

当 webpack-dev-server 开启了热替换后，webpack 会向打包结果中注入 module.hot 属性(module 是在 webpack 替换依赖函数时注入的)。

默认情况下，webpack-dev-server 不管是否开启了热替换，当重新打包后，都会调用浏览器的 location.reload 方法刷新页面。

但如果浏览器运行了 module.hot.accept()，将改变这一行为。module.hot.accept() 会让 webpack-dev-server 通过 socket 管道，把服务器更新的内容发送到浏览器。浏览器收到后将更新的内容交给 HotModuleReplacementPlugin 插件在 webpack 打包后注入的方法进行解析，然后将解析后的代码执行，但并不会改变原始代码。

socket 管道

HotModuleReplacementPlugin 插件为 webpack 自带插件，仅用于开发阶段启动热替换功能。当 webpack 配置文件中设置 devServer.hot 的值为 true 时会自动添加使用该插件，可以不用手动在 webpack 配置文件 plugins 选项中添加。

对于样式的热替换需要借助 style-loader 或 mini-css-extract-plugin 实现。其中 mini-css-extract-plugin 还需要配置 MiniCssExtractPlugin.loader 的 options 选项，详细了解。
