PostCSS

关于 PostCSS

PostCSS 和 LESS、SASS 的思路不同，它只做一些代码分析之类的事情，将分析的结果交给插件，具体的代码转换操作是插件去完成的。

PostCSS 的处理流程有点像 webpack，本身仅做依赖分析、抽象语法树分析，其他的操作是靠插件和加载器完成的。

官网地址: https://postcss.org/
github 地址: https://github.com/postcss/postcss

安装使用 PostCSS

npm i -D postcss

PostCSS 库提供了对应的 api 用于转换代码，如果需要使用 PostCSS 的一些高级功能，或者开发 PostCSS 插件，就要使用 PostCSS 的 api。

PostCSS api 文档地址: http://api.postcss.org/

对于使用者，并不希望使用代码的方式来使用 PostCSS，可以安装 postcss-cli，通过命令行来完成编译。

npm i -D postcss-cli

postcss-cli 提供一个命令，它调用 PostCSS 中的 api 来完成编译。

postcss [源码文件] -o [输出文件]

PostCSS 的配置文件

PostCSS 的配置文件会影响其编译行为。

配置文件的默认名称是 postcss.config.js，配置文件格式与 webpack 相似。

module.exports = {
    map: false, // 关闭 source-map
    plugins: {
        "postcss-preset-env": {
            stage: 0, // 哪怕是处于草案阶段的语法，也需要转换
            preserve: false
        },
        "postcss-apply": {},
        "postcss-color-function": {}
    }
}

PostCSS 的插件

单独使用 PostCSS 没有意义，要让它真正的发挥作用需要插件。

PostCSS 插件市场地址: https://www.postcss.parts/

postcss-preset-env

作为 PostCSS 预设环境，它整合了很多的常用插件，并完成了基本的配置，只需要安装一个插件，就相当于安装了很多插件了。

安装好该插件后，在 PostCSS 配置文件中加入下面的代码完成配置。

module.exports = {
    plugins: {
        "postcss-preset-env": {} // {} 中可以填写插件的配置
    }
}

postcss-preset-env 插件实现以下功能:

  • 可以自动添加厂商前缀

    在旧版本浏览器中，要实现某些新的 CSS 样式，需要使用厂商前缀。postcss-preset-env 内部使用 autoprefixer 库，实现自动添加厂商前缀功能。

    例如:

    ::placeholder {
    color: red;
}

    自动添加厂商前缀后:

    ::-webkit-input-placeholder {
    color: red;
}
::-moz-placeholder {
    color: red;
}
:-ms-input-placeholder {
    color: red;
}
::-ms-input-placeholder {
    color: red;
}
::placeholder {
    color: red;
}

    如果需要调整兼容的浏览器范围，可以通过下面的方式进行配置。

    方式1: 在 postcss-preset-env 的配置文件中加入 browsers

    module.exports = {
    plugins: {
        "postcss-preset-env": {
            browsers: [
                "last 2 version",
                "> 1%"
            ]
        }
    }
}

    方式2【推荐】: 在 .browserslistrc 文件中

    last 2 version
> 1%

    方式3【推荐】: 在 package.json 的配置文件中加入 browserslist

    "browserslist": [
    "last 2 version",
    "> 1%"
]

    默认情况下，匹配的结果求的是并集。关于配置兼容的浏览器范围的规范，详见: https://github.com/browserslist/browserslist 。

    例如:

    last 2 version \\ 浏览器的兼容最近期的两个版本
> 1% in CN \\ 匹配中国大于 1% 的人使用的浏览器
not ie <= 8 \\ 排除掉版本号小于等于 8 的 IE 浏览器

    可以通过网站: https://browserl.ist/ 对配置结果覆盖的浏览器进行查询，查询时，多行之间使用英文逗号分割。

    浏览器兼容范围的数据来自于 CanIUse 网站，由于数据并非实时的，所以不会特别准确。

  • 可以使用未来的 CSS 语法

    CSS 的某些前沿语法正在制定过程中，没有形成真正的标准，要使用这部分语法，为了浏览器兼容性，需要进行编译。

    postcss-preset-env 内部使用 cssnext 库，支持使用未来的 CSS 语法功能。

    通过 postcss-preset-env 的 stage 配置，告知需要对哪个阶段的 CSS 语法进行兼容处理，默认值为 2。

    "postcss-preset-env": {
    stage: 0
}

    这里一共有五个阶段可配置:

    • Stage 0: Aspirational - 只是一个早期草案，极其不稳定。
    • Stage 1: Experimental - 仍然极其不稳定，但是提议已被 W3C 公认。
    • Stage 2: Allowable - 虽然还是不稳定，但已经可以使用了。
    • Stage 3: Embraced - 比较稳定，可能将来会发生一些小的变化，它即将成为最终的标准。
    • Stage 4: Standardized - 所有主流浏览器都应该支持的 W3C 标准。

    关于未来的 CSS 语法:

    • 变量

      在 :root{} 中定义常用变量，并使用 -- 前缀命名变量。

      :root {
    --lightColor: #ddd;
    --darkColor: #333;
}

a {
    color: var(--lightColor);
    background: var(--darkColor);
}

      编译后仍然可以看到原语法，因为某些新语法的存在并不会影响浏览器的渲染，尽管浏览器可能不认识。

      如果不希望在结果中看到新语法，可以配置 postcss-preset-env 的 preserve 为 false。

    • 自定义选择器

      @custom-selector :--heading h1, h2, h3, h4, h5, h6;
@custom-selector :--enter :focus,:hover;

a:--enter {
    color: #f40;
}

:--heading {
    font-weight:bold;
}

:--heading.active {
    font-weight:bold;
}

      编译后:

      a:focus,a:hover {
    color: #f40;
}

h1,h2,h3,h4,h5,h6 {
    font-weight:bold;
}

h1.active,h2.active,h3.active,h4.active,h5.active,h6.active {
    font-weight:bold;
}

    • 嵌套

      嵌套的选择器前必须使用符号 &。

      .a {
    color: red;
    & .b {
        color: green;
    }

    & > .b {
        color: blue;
    }

    &:hover {
        color: #000;
    }
}

      编译后:

      .a {
    color: red;
}

.a .b {
    color: green;
}

.a>.b {
    color: blue;
}

.a:hover {
    color: #000;
}

postcss-apply

该插件支持在 CSS 中书写属性集。可以利用 CSS 的新语法定义一个 CSS 代码片段，然后在需要的时候应用它。

:root {
    --center: {
        position: absolute;
        left: 50%;
        top: 50%;
        transform: translate(-50%, -50%);
    }
}

.item {
    @apply --center;
}

编译后:

.item {
    position: absolute;
    left: 50%;
    top: 50%;
    -webkit-transform: translate(-50%, -50%);
    transform: translate(-50%, -50%);
}

postcss-color-function

该插件支持在源码中使用一些颜色函数。

body {
    /* 使用颜色#aabbcc，不做任何处理，等同于直接书写 #aabbcc */
    color: color(#aabbcc);
    /* 将颜色#aabbcc透明度设置为90% */
    color: color(#aabbcc a(90%));
    /* 将颜色#aabbcc的红色部分设置为90% */
    color: color(#aabbcc red(90%));
    /* 将颜色#aabbcc调亮50%（更加趋近于白色），类似于less中的lighten函数 */
    color: color(#aabbcc tint(50%));
    /* 将颜色#aabbcc调暗50%（更加趋近于黑色），类似于less中的darken函数 */
    color: color(#aabbcc shade(50%));
}

编译后:

body {
    /* 使用颜色#aabbcc，不做任何处理，等同于直接书写 #aabbcc */
    color: rgb(170, 187, 204);
    /* 将颜色#aabbcc透明度设置为90% */
    color: rgba(170, 187, 204, 0.9);
    /* 将颜色#aabbcc的红色部分设置为90% */
    color: rgb(230, 187, 204);
    /* 将颜色#aabbcc调亮50%（更加趋近于白色），类似于less中的lighten函数 */
    color: rgb(213, 221, 230);
    /* 将颜色#aabbcc调暗50%（更加趋近于黑色），类似于less中的darken函数 */
    color: rgb(85, 94, 102);
}

postcss-import

该插件支持在 PostCSS 文件中导入其他样式代码，通过该插件可以将它们合并。

将 PostCSS 加入到 webpack 中，而 webpack 本身具有依赖分析的功能，所以该插件的实际意义不大。

stylelint

该插件可以即时的发现错误。

stylelint 官网地址: https://stylelint.io/

插件本身并没有提供具体的规则验证，所以需要安装或自行编写规则验证方案，通常安装 stylelint-config-standard 库来提供标准的 CSS 规则判定。

安装好后可使用文件 .stylelintrc，告知 stylelint 使用该库来进行规则验证。

{
    "extends": "stylelint-config-standard"
}

此时，如果代码出现不规范的地方，编译时将会报出错误，例如:

body {
    background: #f4;
}

如果某些规则并非所期望的，可以在配置中通过 rules 进行设置。

{
    "extends": "stylelint-config-standard",
    "rules": {
        "indentation": null \\ 禁用该规则
    }
}

该插件除了在 PostCSS 中使用，在编译时报告错误。也可以在编辑器里使用，在编辑时报告错误。通过 vscode 中安装 stylelint 插件，即可读取工程中的配置文件，按照配置进行实时报错。
