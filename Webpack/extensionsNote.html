webpack 常用扩展

clean-webpack-plugin

一个 webpack 插件，用来删除/清理你的构建文件夹。

默认情况下，这个插件会删除 webpack 输出中的所有文件。路径目录，以及每次成功重新构建后所有未使用的 webpack 资产。

安装：npm i -D clean-webpack-plugin

const { CleanWebpackPlugin } = require('clean-webpack-plugin');

module.exports = {
    plugins: [
        new CleanWebpackPlugin()
    ]
}

html-webpack-plugin

简化 HTML 文件创建的插件。

这是一个 webpack 插件，可以简化 HTML 文件的创建，从而为您的 webpack bundle 提供服务。这对于 webpack bundle 特别有用，因为它在文件名中包含一个哈希值，这会改变每次编译。可以使用 lodash 语法编写你自己的模板，或者使用你自己的加载器，让插件为你生成一个 HTML 文件。

安装：npm i -D html-webpack-plugin

const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
    plugins: [
        new HtmlWebpackPlugin()
    ]
}

copy-webpack-plugin

将已经存在(静态)的单个文件或整个目录复制到构建目录。

安装：npm i -D copy-webpack-plugin

const CopyPlugin = require('copy-webpack-plugin');

module.exports = {
    plugins: [
        new CopyPlugin({
            patterns: [
                { from: 'source', to: 'dist' }
            ]
        })
    ]
}

file-loader

将文件中的 import/require() 解析为一个 url ，并将该文件发送到输出目录中。

安装：npm i -D file-loader

module.exports = {
    module: {
        rules: [
            {
                test: /\.(png|jpe?g|gif)$/i,
                use: [
                    {
                        loader: 'file-loader'
                    }
                ]
            }
        ]
    }
}

url-loader

一个 webpack 加载器，它将文件转换为 base64 url 。

工作原理类似于 file-loader ，但是如果文件小于字节限制，则可以返回 DataURL 。

安装：npm i -D url-loader

module.exports = {
    module: {
        rules: [
            {
                test: /\.(png|jpg|gif)$/i,
                use: [
                    {
                        loader: 'url-loader',
                        options: {
                            limit: 8192
                        }
                    }
                ]
            }
        ]
    }
}

css-loader

CSS 加载器像 import / require() 一样解释 @import 和 url()，并解析它们。css-loader 用于解决 CSS 文件细分问题。

css-loader 的作用是将 CSS 代码转换为 js 代码。其工作原理可以理解为，首先将 CSS 文件的内容作为字符串导出，再将 CSS 中的其他依赖作为 require 导入，以便 webpack 分析依赖。

安装：npm i -D css-loader

css-loader 提供 modules{Boolean|String|Object} 选项，用于启用/禁用 CSS 模块规范和设置基本行为。css modules 用于解决 CSS 类名冲突问题。[1]

module.exports = {
    module: {
        rules: [
            {
                test: /\.css$/i,
                use: ['style-loader', 'css-loader']
            }
        ]
    }
}

style-loader

将 css-loader 处理好的结果注入到 DOM 中。

因 webpack 在构建依赖模块时，会跳过已经分析过的文件，而直接取其之前的结果。所以 style-loader 有能力避免同一个样式的重复导入。

安装：npm i -D style-loader

module.exports = {
    module: {
        rules: [
            {
                test: /\.css$/i,
                use: ['style-loader', 'css-loader']
            }
        ]
    }
}

less-loader

一个 webpack 加载器，它将 less 文件编译成 CSS 文件。less 预编译器用于解决 CSS 重复样式的问题。[2]

安装：npm i -D less-loader

module.exports = {
    module: {
        rules: [
            {
                test: /\.less$/,
                loader: 'less-loader'
            }
        ]
    }
}

postcss-loader

一个 webpack 加载器，它将使用 PostCSS 处理 CSS。[3]

安装：npm i -D postcss-loader

module.exports = {
    module: {
        rules: [
            {
                test: /\.pcss$/,
                use: [ 'style-loader', 'css-loader', 'postcss-loader' ]
            }
        ]
    }
}

mini-css-extract-plugin

这个插件将 CSS 提取到单独的文件中。它为每个包含 CSS 的 JS 文件创建一个 CSS 文件。通常每个 chunk 对应一个 CSS 文件。它支持按需加载 CSS 和 SourceMaps。

该插件提供了1个 plugin 负责生成 CSS 文件和1个 loader 负责记录要生成的 CSS 文件的内容，同时导出开启 css-module 后的样式对象。

安装：npm i -D mini-css-extract-plugin

const MiniCssExtractPlugin = require('mini-css-extract-plugin');

module.exports = {
    plugins: [new MiniCssExtractPlugin()],
    module: {
        rules: [
            {
                test: /\.css$/i,
                use: [
                    {
                        loader: MiniCssExtractPlugin.loader,
                        options: {
                            esModule: true
                        }
                    },
                    'css-loader'
                ]
            }
        ]
    }
}

babel-loader

这个包允许使用 Babel 和 webpack 编译 JavaScript 文件。babel-loader 用于解决 JS 兼容性的问题。[4]

安装：npm i -D babel-loader @babel/core

module.exports = {
    module: {
        rules: [
            {
                test: /\.js$/,
                exclude: /node_modules/,
                use: "babel-loader"
            }
        ]
    }
}

cache-loader

一个 webpack 加载器，允许将后续加载器的结果缓存到磁盘(默认)或数据库中。cache-loader 用于优化构建性能，详细了解。

使用时需要将这个加载器添加到其他(昂贵的)加载器之前，以便在磁盘上缓存结果。请注意，保存读取和保存缓存文件会有开销，因此只使用这个加载器来缓存昂贵的加载器。

安装：npm i -D cache-loader

module.exports = {
    module: {
        rules: [
            {
                test: /\.js$/,
                use: ['cache-loader', `babel-loader`],
                include: path.resolve('src')
            }
        ]
    }
}

thread-loader

一个 webpack 加载器，允许将后续加载器在一个新线程中运行。thread-loader 用于优化构建性能，详细了解。

使用时需要将这个加载器添加到其他(昂贵的)加载器之前，后续的加载器将在新的线程中运行。请注意，每个线程都是一个独立的 nodejs 进程，开销约为 600ms，并且还有进程间通信的开销，因此只对昂贵的加载器使用这个加载器。

安装：npm i -D thread-loader

module.exports = {
    module: {
        rules: [
            {
                test: /\.js$/,
                use: ['thread-loader', `babel-loader`],
                include: path.resolve('src')
            }
        ]
    }
}

const TerserPlugin = require('terser-webpack-plugin');

}

optimize-css-assets-webpack-plugin

一个 webpack 插件，它将在 webpack 构建期间搜索 CSS 资产，并优化\最小化 CSS。默认情况下，它使用 cssnano，也可以指定一个定制的 CSS 处理器。optimize-css-assets-webpack-plugin 用于优化传输性能，详细了解。

css-minimizer-webpack-plugin 插件就像 optimize-css-assets-webpack-plugin 一样使用 cssnano 来优化和缩小 CSS，但是它能更准确的使用查询字符串的源映射和资产，允许缓存并以并行模式工作。

安装：npm i -D optimize-css-assets-webpack-plugin

const OptimizeCssAssetsPlugin = require('optimize-css-assets-webpack-plugin');

module.exports = {
    optimization: {
        minimize: true,
        minimizer: [new OptimizeCssAssetsPlugin()]
    }
}

这是一个完善 Tree Shaking 深度作用域分析的 webpack 插件。它可以使 webpack 消除与未使用的导出相关的未使用的导入。webpack-deep-scope-plugin 用于优化传输性能，详细了解，不过这个插件缺少维护。

安装：npm i -D webpack-deep-scope-plugin

const WebpackDeepScopeAnalysisPlugin = require('webpack-deep-scope-plugin').default;

module.exports = {
    plugins: [
        new WebpackDeepScopeAnalysisPlugin()
    ]
}

purgecss-webpack-plugin

这是一个删除未使用的 CSS 的 webpack 插件。purgecss-webpack-plugin 用于优化传输性能，详细了解。

安装：npm i -D purgecss-webpack-plugin

const Purgecss = require("purgecss-webpack-plugin");
const path = require("path");
const globAll = require("glob-all");
const srcAbs = path.resolve(__dirname, "src");
const htmlPath = path.resolve(__dirname, "public/index.html");
const paths = globAll.sync([`${srcAbs}**/*.js`, htmlPath]);

module.exports = {
    plugins: [
        new Purgecss({
            paths
        })
    ]
}

webpack-merge

可用于合并配置对象的 webpack 工具。

通过提供一个 merge 函数，用于连接数组和合并对象，创建一个新对象。如果遇到函数，它将执行它们，通过算法运行结果，然后再次将返回值包装到函数中。

安装：npm i -D webpack-merge

const baseConfig = require("./config/webpack.base");
const developmentConfig = requie("./config/webpack.development");
const productionConfig = require("./config/webpack.production");
const merge = require("webpack-merge");

module.exports = mode => {
    if(mode === "production"){
        return merge(baseConfig,productionConfig);
    }
    return merge(baseConfig,developmentConfig);
}
