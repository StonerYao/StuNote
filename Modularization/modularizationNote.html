模块化笔记

CommonJS

CommonJS导出

exports 是一个空的对象，可以为该对象添加任何需要导出的内容

exports: {
    abc: 123
}

exports.cba = fn

/*
exports: {
    abc: 123,
    cba: fn
}
*/

CommonJS导入

require 是一个函数，传入模块的路径即可返回该模块导出的整个内容

require("./test.js")

/*
{abc: 123, cba: fn}
*/

require("./abc.js") 是在当前目录下找abc.js文件
require("abc.js") 是在node_modules文件夹下找相应入口文件

nodejs对CommonJS的实现

为了实现CommonJS规范，nodejs对模块做出了以下处理

  • 为了保证高效的执行，仅加载必要的模块。nodejs只有执行到require函数时才会加载并执行模块

  • 为了隐藏模块中的代码，nodejs执行模块时，会将模块中的所有代码放置到一个函数中执行，以保证不污染全局变量。

     (function(){
     //模块中的代码
 })()

  • 为了保证顺利的导出模块内容，nodejs做了以下处理

    • 在模块开始执行前，初始化一个值module.exports = {}
    • module.exports即模块的导出值
    • 为了方便开发者便捷的导出，nodejs在初始化完module.exports后，又声明了一个变量exports = module.exports
     (function(module){
     module.exports = {};
     var exports = module.exports;
     //模块中的代码
     return module.exports;
 })()

  • 为了避免反复加载同一个模块，nodejs默认开启了模块缓存，如果加载的模块已经被加载过了，则会自动使用之前的导出结果

  • 可以认为，CommonJS是同步的，必须要等到加载完文件并执行完代码后才能继续向后执行

ES6模块化

ES6模块化导出

每个模块，除了允许有多个基本导出之外，还允许有一个默认导出

  • 基本导出

    export 声明表达式 或 export {具名符号}

    • 基本导出可以有多个，每个必须有名称
    export var a = 1;

var b = 2;
export { b };

var c = 3;
export { c as abc };

  • 默认导出

    export default 默认导出的数据 或 export {默认导出的数据 as default}

    • 每个模块仅允许有一个默认导出
    export default {
  fn: function () {},
  a: "abc"
}

var b = 123;
export { b as default}

ES6模块化导入

使用依赖预声明的方式导入模块，导入任何其他模块，导入代码必须放置到所有代码之前

如果多次导入一个相同的模块，那这个被导入的模块只会被运行一次

  • 基本导入

    import {导入的符号列表} from "模块路径"

    • 规范的路径表示法：所有路径必须以./或../开头
    • 导入时可以通过关键字as对导入的符号进行重命名
    • 导入时使用的符号是常量，不可修改
    import { a , b } from "./test.js";

import { c as d }from "./test.js"

//b = 123; 这是错误的

console.log(a, b, d)

  • 默认导入

    import 接收变量名 from "模块路径"

    • 由于默认导入时变量名是自行定义的，因此没有别名一说
    import a from "./test.js"
}

  • 基本导入 + 默认导入

    import 接收默认导出的变量, {接收基本导出的变量} from "模块路径"
    或
    import * as 接收对象名 from "模块路径"

    • 可以使用*号导入所有的基本导出和默认导出，会将所有基本导出和默认导出聚合到一个对象中，默认导出会作为属性default存在
    import data, { a, b } from "./test.js"
//另一种写法 import { a, b, default as data } from "./test.js"

import * as data from "./test.js"

ES6模块化注意事项

  • 尽量导出不可变值

当导出一个内容时，尽量保证该内容是不可变的（大部分情况都是如此）

因为，虽然导入后，无法更改导入内容，但是在导入的模块内部却有可能发生更改，这将导致一些无法预料的事情发生

所以，在导入的模块中使用 const 常量导出

  • 可以使用无绑定的导入用于执行一些初始化代码

如果我们只是想执行模块中的一些代码，而不需要导入它的任何内容，可以使用无绑定的导入：

import "模块路径"
//例如 import "./test.js"
  • 可以使用绑定再导出，来重新导出来自另一个模块的内容

有的时候，我们可能需要用一个模块封装多个模块，然后有选择的将多个模块的内容分别导出，可以使用下面的语法轻松完成

export {绑定的标识符} from "模块路径"

浏览器对ES6模块化的实现

  • 浏览器使用以下方式引入一个ES6模块文件

    <script src="入口文件" type="module">
