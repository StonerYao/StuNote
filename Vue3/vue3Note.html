全局 API

createApp

在 Vue2 中，没有把组件实例和 Vue 应用两个概念区分开，通过构造函数 new Vue() 创建的 Vue 实例对象，既是一个 Vue 应用，同时又是一个特殊的 Vue 组件。

Vue3 不存在构造函数 Vue，而是通过一个新的全局 API: createApp 创建 Vue 应用，并返回一个提供应用上下文的应用实例。应用实例挂载的整个组件树共享相同的上下文，该上下文提供了之前在 Vue2 中全局的配置。

import { createApp } from 'vue'

const app = createApp({
  // 接收一个根组件选项对象作为第一个参数
  data() {
    return {
      ...
    }
  },
  methods: {...},
  computed: {...}
  ...
})

Vue3 真正实现了把组件实例和 Vue 应用两个概念区分开，通过 createApp 创建的 Vue 实例对象，就是一个 Vue 应用。它内部提供的方法是针对整个应用的，而不再是一个特殊的组件。

h

在 Vue3 中，h (createElement) 函数是全局导入的，而不是作为参数自动传递给 render 函数。

import { h } from 'vue'

export default {
  render() {
    return h('div')
  }
}

Vue3 的 render 函数不再接收任何参数，主要用于 setup() 函数内部。并且可以访问作用域中声明的响应式状态和函数，以及传递给 setup() 的参数。

import { h, reactive } from 'vue'

export default {
  setup(props, { slots, attrs, emit }) {
    const state = reactive({
      count: 0
    })

    function increment() {
      state.count++
    }

    // 返回render函数
    return () =>
      h(
        'div',
        {
          onClick: increment
        },
        state.count
      )
  }
}

Vue3 中 VNode props 结构是扁平的。

// Vue2
{
  class: ['button', 'is-outlined'],
  style: { color: '#34495E' },
  attrs: { id: 'submit' },
  domProps: { innerHTML: '' },
  on: { click: submitForm },
  key: 'submit-button'
}

// Vue3
  class: ['button', 'is-outlined'],
  style: { color: '#34495E' },
  id: 'submit',
  innerHTML: '',
  onClick: submitForm,
  key: 'submit-button'
}

defineAsyncComponent

在 Vue3 中，由于函数式组件被定义为纯函数，因此异步组件需要通过新的 defineAsyncComponent 助手方法来显式地定义，defineAsyncComponent 可以接受一个返回 Promise 的工厂函数。

并且 component 选项现在被重命名为 loader，以便准确地传达不能直接提供组件定义的信息。

// 局部注册

import { createApp, defineAsyncComponent } from 'vue'
import ErrorComponent from './components/ErrorComponent.vue'
import LoadingComponent from './components/LoadingComponent.vue'

// 不带选项的异步组件
const asyncPage = defineAsyncComponent(() => import('./NextPage.vue'))

// 带选项的异步组件
const asyncPageWithOptions = defineAsyncComponent({
  // 工厂函数
  loader: () => import('./NextPage.vue'),
  // 在显示 loadingComponent 之前的延迟 | 默认值：200（单位 ms）
  delay: 200,
  // 如果提供了 timeout ，并且加载组件的时间超过了设定值，将显示错误组件
  // 默认值：Infinity（即永不超时，单位 ms）
  timeout: 3000,
  // 加载失败时要使用的组件
  errorComponent: ErrorComponent,
  // 加载异步组件时要使用的组件
  loadingComponent: LoadingComponent,
  /**
   *
   * @param {*} error 错误信息对象
   * @param {*} retry 一个函数，用于指示当 promise 加载器 reject 时，加载器是否应该重试
   * @param {*} fail  一个函数，指示加载程序结束退出
   * @param {*} attempts 允许的最大重试次数
   */
  onError(error, retry, fail, attempts) {
    if (error.message.match(/fetch/) && attempts <= 3) {
      // 请求发生错误时重试，最多可尝试 3 次
      retry()
    } else {
      // 注意，retry/fail 就像 promise 的 resolve/reject 一样：
      // 必须调用其中一个才能继续错误处理。
      fail()
    }
  }
})

createApp({
  // ...
  components: {
    asyncPage,
    asyncPageWithOptions
  }
})

loader 函数不再接收 resolve 和 reject 参数，且必须始终返回 Promise。在返回 Promise 的工厂函数中，resolve 回调应该在服务端返回组件定义后被调用，reject 回调应该在加载失败时被调用。

// 全局注册

import { createApp, defineAsyncComponent } from 'vue'

const app = createApp({})

const AsyncComp = Vue.defineAsyncComponent(
  () =>
    new Promise((resolve, reject) => {
      resolve({
        template: '<div>I am async!</div>'
      })
    })
)

app.component('async-example', AsyncComp)

应用 API

在 Vue2 中，同一个构造函数创建的每个根实例共享相同的全局配置，也就是说调用构造函数的静态方法 (例如: Vue.use()) 会全局改变 Vue 的行为，这样不利于隔离不同应用。同时构造函数中集成了太多的功能，也不利于 Tree Shaking。

在 Vue3 中，通过 createApp(/* options */) 创建应用实例后，即可使用这个实例上的方法，例如通过 use 使用插件，通过 mount 挂载 App 实例等等，并且这些方法可以链式调用。

import { createApp } from 'vue'
import MyApp from './MyApp.vue'
import VueRouter from './VueRouter'

// mount 方法返回根组件实例
const vm = createApp(MyApp).use(VueRouter).mount('#app')

Vue3 将任何全局改变 Vue 行为的 API 移动到应用实例上，当页面有多个 Vue 应用时，只会影响特定的应用实例，不会污染其它的 Vue 应用。另外这些功能使用普通函数导出，能够充分利用 Tree Shaking 优化打包体积。

选项式 API

emits

Vue3 新增了 emits 选项，类似于 props 选项，用于定义组件可以向其父对象发出的事件。

emits 选项可以是数组或对象，使用对象语法允许为与发出的事件一起传递的参数定义验证器。验证器将为事件分配一个函数，该函数接收传递给 $emit 调用的参数，并返回一个布尔值以指示事件是否有效。

<template>
  <div>
    <p>{{ text }}</p>
    <button v-on:click="$emit('accepted')">OK</button>
  </div>
</template>
<script>
  export default {
    props: ['text'],
    emits: ['accepted']
  }
</script>

Vue3 去除了 v-on 的 .native 修饰符。当在 emits 选项中定义了原生事件 (如 click) 时，将使用组件中的事件替代原生事件侦听器。

默认情况下 (除非在组件的选项中设置了 inheritAttrs: false)，未在 emits 中声明的事件侦听器都将作为原生事件侦听器添加到组件的根元素中，且绑定到组件的 $attrs property 中。而 emits 选项中列出的事件则不会被组件的根元素继承，也将从 $attrs property 中移除。建议在每个组件中使用 emits 声明发出的所有事件，以便更好地记录组件应该如何工作。

实例 API

在创建了实例后，实例上提供了以下属性和方法，这些属性和方法都有一个 $ 前缀，以避免与用户定义的属性和方法名冲突:

属性: https://v3.cn.vuejs.org/api/instance-properties.html
方法: https://v3.cn.vuejs.org/api/instance-methods.html

同时在实例的选项对象中，也可以通过 this 访问到实例的属性和方法。

但与 Vue2 不同的是，Vue3 创建的实例以及实例选项对象中的 this 并不是实例对象本身，而是一个 Proxy (代理对象)，实际上实例的属性和方法都是由它提供的。

指令

v-model

在 Vue3 中，当对自定义组件使用 v-model 指令时，绑定的属性名由原来的 value 变为 modelValue，事件名由原来的 input 变为 update:modelValue。

<ChildComponent v-model="pageTitle" />

<!-- 前者是后者的简写 -->

<ChildComponent
  :modelValue="pageTitle"
  @update:modelValue="pageTitle = $event"
/>

Vue3 移除了 v-bind 的 .sync 修饰符和组件的 model 选项，而由传参的 v-model 替代。

<ChildComponent :title.sync="pageTitle" />

<!-- 前者需替换为后者 -->

<ChildComponent v-model:title="pageTitle" />

<!-- 前者是后者的简写 -->

<ChildComponent :title="pageTitle" @update:title="pageTitle = $event" />

Vue3 允许在同一个组件上使用多个 v-model 进行双向绑定。

<ChildComponent v-model:title="pageTitle" v-model:content="pageContent" />

<!-- 前者是后者的简写 -->

<ChildComponent
  :title="pageTitle"
  @update:title="pageTitle = $event"
  :content="pageContent"
  @update:content="pageContent = $event"
/>

Vue3 除了内置的修饰符 (.trim、.number 和 .lazy) 以外，还允许自定义 v-model 修饰符。在 Vue3 中，添加到组件 v-model 的修饰符将通过 modelModifiers prop 在组件的 created 生命周期钩子触发时提供给组件。

<my-component v-model.capitalize="myText"></my-component>

const app = Vue.createApp({
  data() {
    return {
      myText: ''
    }
  }
})

app.component('my-component', {
  props: {
    modelValue: String,
    modelModifiers: {
      default: () => ({})
    }
  },
  emits: ['update:modelValue'],
  methods: {
    emitValue(e) {
      let value = e.target.value
      if (this.modelModifiers.capitalize) {
        value = value.charAt(0).toUpperCase() + value.slice(1)
      }
      this.$emit('update:modelValue', value)
    }
  },
  template: `<input
    type="text"
    :value="modelValue"
    @input="emitValue">`
})

app.mount('#app')

对于带参数的 v-model 绑定，生成的 prop 名称将为 arg + "Modifiers"。

自定义修饰符

v-if & v-for

在 Vue3 中，v-if 与 v-for 作用于同一个元素上时，v-if 会拥有比 v-for 更高的优先级，这意味着 v-if 将没有权限访问 v-for 里的变量。

由于语法上存在歧义，要避免在同一元素上同时使用 v-if 与 v-for。可以把 v-for 移动到 <template> 标签中来修正，更好的办法是通过创建计算属性筛选出列表，并以此创建可见元素。

<template v-for="(item, index) in sellingsRef" :key="item.id">
  <span>{{ index + 1 }}.</span>
  <strong>{{ item.title }}</strong>
</template>

setup() {
  const tasksRef = ref([
    { id: 1, sell: true, title: "iphone12" },
    { id: 2, sell: false, title: "xiaomi" },
    { id: 3, sell: true, title: "huawei" },
    { id: 4, sell: true, title: "vivo" },
  ]);
  const sellingsRef = computed(() => tasksRef.value.filter((it) => it.sell));
  return {
    tasksRef,
    sellingsRef,
  };
}

特性

key

在 Vue3 中使用特性 key 时，需要注意如下的改变:

当使用 v-if、v-else-if、v-else 分支的时候，不再需要指定 key 值，Vue3 会自动生成唯一的 key 给予每个分支。即便要手动给予 key 值，也必须让每个分支拥有唯一的 key，重复的 key 会造成渲染错误。

<!-- vue2 -->
<div v-if="condition" key="yes">Yes</div>
<div v-else key="no">No</div>

<!-- vue3 -->
<div v-if="condition">Yes</div>
<div v-else>No</div>

当使用 <template> 进行 v-for 循环时，需要把 key 值放到 <template> 中，而不是它的子元素中。

<!-- vue2 -->
<template v-for="item in list">
  <div :key="item.id">...</div>
  <span :key="item.id">...</span>
</template>

<!-- vue3 -->
<template v-for="item in list" :key="item.id">
  <div>...</div>
  <span>...</span>
</template>

内置组件

teleport

Vue3 中新增了 Teleport 内置组件，它提供了一种简洁的方式，让我们能够控制在 DOM 中的哪个父节点下渲染一段 HTML，而不必求助于全局状态或将其分割成两个组件。

Props to - string 用于指定移动 Teleport 插槽内容的目标元素，且必须是有效的 CSS 选择器。

Props disabled - boolean 用于禁用 Teleport 的功能。Teleport 插槽内容将在原始的位置渲染。

注意: 这将移动实际的 DOM 节点，而不是被销毁和重新创建，并且还将保持任何组件实例的活动状态。所有有状态的 HTML 元素 (例如播放的视频) 都将保持其状态。

<body>
  <div style="position: relative;">
    <h3>Tooltips with Vue 3 Teleport</h3>
    <div>
      <modal-button></modal-button>
    </div>
  </div>
</body>

app.component('modal-button', {
  template: `
    <button @click="modalOpen = true">
      Open full screen modal! (With teleport!)
    </button>

    <teleport to="body">
      <div v-if="modalOpen" class="modal">
        <div>
          I'm a teleported modal! 
          (My parent is "body")
          <button @click="modalOpen = false">
            Close
          </button>
        </div>
      </div>
    </teleport>
  `,
  data() {
    return { 
      modalOpen: false
    }
  }
})

当 Teleport 包含 Vue 组件时，则被包含的组件仍将是 Teleport 父组件的逻辑子组件。所以来自父组件的注入将按预期工作，并且子组件将嵌套在 Vue Devtools 中的父组件之下，而不是放在实际内容移动到的位置。

const app = Vue.createApp({
  template: `
    <h1>Root instance</h1>
    <parent-component />
  `
})

app.component('parent-component', {
  template: `
    <h2>This is a parent component</h2>
    <teleport to="#endofbody">
      <child-component name="John" />
    </teleport>
  `
})

app.component('child-component', {
  props: ['name'],
  template: `
    <div>Hello, {{ name }}</div>
  `
})

可以在同一目标上使用多个 Teleport，并按照先后顺序挂载到目标元素上。

<teleport to="#modals">
  <div>A</div>
</teleport>
<teleport to="#modals">
  <div>B</div>
</teleport>

<!-- result-->
<div id="modals">
  <div>A</div>
  <div>B</div>
</div>

组合式 API

组合式 API 可以让同一个逻辑关注点相关的代码聚合在一起，从而有机会更好地理解和维护复杂的组件。对于不同组件的相同功能，也能够更好的复用。同时也有利于和类型推断系统 (如 TS) 深度配合。

setup

setup 组件选项作为组合式 API 的入口点，会在 props 解析完成后以及创建组件之前 (所有生命周期钩子函数之前) 自动执行。

注意: 由于在执行 setup 时尚未创建组件实例，因此在 setup 选项中 this 指向 undefined。这意味着，除了 props、attrs、slots 和 emit 之外，将无法访问组件中声明的任何属性 (data、computed、methods)。

从 setup 返回的所有内容都将暴露给组件的其余部分 (计算属性、方法、生命周期钩子等等) 以及组件的模板。

<template>
  <div>{{ plusOne }}</div>
</template>

<script>
  import { ref, computed } from 'vue'

  export default {
    setup() {
      const count = ref(0)
      const plusOne = computed(() => count.value + 1)

      return {
        count,
        plusOne
      }
    }
  }
</script>

注意: 从 setup 返回的 refs 在模板中访问时是被自动解开的，因此不应在模板中使用 .value。

同时 setup 还可以返回一个渲染函数，该函数可以直接使用在同一作用域中声明的响应式状态。

import { h, ref, reactive } from 'vue'

export default {
  setup() {
    const readersNumber = ref(0)
    const book = reactive({ title: 'Vue 3 Guide' })
    return () => h('div', [readersNumber.value, book.title])
  }
}

setup 函数可以接受两个参数，第一个参数是 props，用于接收来自父组件的数据。它是响应式的，当传入新的 prop 时，它将被更新。

export default {
  props: {
    title: String
  },
  setup(props) {
    console.log(props.title)
  }
}

注意: props 不能使用 ES6 解构，因为它会消除 prop 的响应性。不过可以通过使用 setup 函数中的 toRefs 来完成此操作。另外 props 对象在开发过程中对于用户区代码是不可变的 (如果用户代码尝试对其进行更改，则会发出警告)。

传递给 setup 函数的第二个参数是 context，用于暴露组件的 attrs、slots、emit 三个 property。context 是一个普通的 JavaScript 对象，它不是响应式的，所以可以使用 ES6 解构。attrs 和 slots 是内部组件实例上相应值的代理，这样可以确保它们即使在更新后也始终会显示最新值。

成员 类型 说明
attrs 对象 同 Vue2 的 this.$attrs
slots 对象 同 Vue2 的 this.$slots
emit 方法 同 Vue2 的 this.$emit 
const MyComponent = {
  setup(props, { attrs }) {
    // 稍后可能会调用的函数
    function onClick() {
      console.log(attrs.foo) // 保证是最新引用
    }
  }
}

生命周期钩子

在组合式 API 中，通过在生命周期钩子前面加上 on 来访问组件的生命周期钩子。并且这些生命周期钩子注册函数只能在 setup() 期间同步使用，因为它们依赖于内部全局状态来定位当前活动实例。

Vue2 选项式 API Vue3 选项式 API Vue3 组合式 API
beforeCreate beforeCreate - 删
created created - 删
beforeMount beforeMount onBeforeMount
mounted mounted onMounted
beforeUpdate beforeUpdate onBeforeUpdate
updated updated onUpdated
beforeDestroy beforeUnmount 改 onBeforeUnmount
destroyed unmounted 改 onUnmounted
errorCaptured errorCaptured onErrorCaptured
- renderTracked 增 onRenderTracked
- renderTriggered 增 onRenderTriggered

注意:

  • Vue3 组合式 API 中，因为 setup 是围绕 beforeCreate 和 created 生命周期钩子运行的，所以不需要显式地定义它们。换句话说，在这些钩子中编写的任何代码都应该直接在 setup 函数中编写。
  • Vue3 中，原 destroyed 和 beforeDestroy 生命周期选项被重命名，使用和功能不变。
  • Vue3 中，新增 renderTracked 和 renderTriggered 生命周期选项用于开发时的调试。

renderTracked 生命周期选项，用于跟踪组件的渲染时的依赖以及依赖的目标对象和键。钩子接收 DebuggerEvent 作为参数，并在虚拟 DOM 渲染时调用。

renderTriggered 生命周期选项，用于跟踪触发了重新渲染的操作以及操作的目标对象和键。钩子接收 DebuggerEvent 作为参数，并在虚拟 DOM 重新渲染时调用。

DebuggerEvent:

  • target: 跟踪或触发渲染的对象。
  • key: 跟踪或触发渲染的属性。
  • type: 跟踪或触发渲染的方式。
<div id="app">
  <button v-on:click="addToCart">Add to cart</button>
  <p>Cart({{ cart }})</p>
</div>

const app = Vue.createApp({
  data() {
    return {
      cart: 0
    }
  },
  renderTracked({ key, target, type }) {
    console.log({ key, target, type })
    /* 当组件第一次渲染时，这将被记录下来:
    {
      key: "cart",
      target: {
        cart: 0
      },
      type: "get"
    }
    */
  },
  renderTriggered({ key, target, type }) {
    console.log({ key, target, type })
  },
  methods: {
    addToCart() {
      this.cart += 1
      /* 这将导致 renderTriggered 调用
        {
          key: "cart",
          target: {
            cart: 1
          },
          type: "set"
        }
      */
    }
  }
})

app.mount('#app')

生命周期钩子函数接受一个回调函数，当钩子被组件调用时将会被执行。在生命周期钩子内同步创建的侦听器和计算属性会在组件卸载时自动删除，因为组件实例上下文也是在生命周期钩子的同步执行期间设置的。

import { onMounted, onUpdated, onUnmounted } from 'vue'

const MyComponent = {
  setup() {
    onMounted(() => {
      console.log('mounted!')
    })
    onUpdated(() => {
      console.log('updated!')
    })
    onUnmounted(() => {
      console.log('unmounted!')
    })
  }
}

Provide / Inject

在组合式 API 中使用 provide/inject 依赖注入。两者都只能在当前活动实例的 setup() 期间调用。

在 setup() 中使用 provide 时，首先要显式导入 provide 方法，再通过调用 provide 方法来定义需要依赖的 property。provide 函数接收两个参数，第一个参数是 property 的键 (String 类型)，第二个参数是 property 的值。

<template>
  <MyMarker />
</template>

<script>
import { provide } from 'vue'
import MyMarker from './MyMarker.vue

export default {
  components: {
    MyMarker
  },
  setup() {
    provide('location', 'North Pole')
    provide('geolocation', {
      longitude: 90,
      latitude: 135
    })
  }
}
</script>

在 setup() 中使用 inject 时，首先要显式导入 inject 方法，再通过调用 inject 方法来定义需要注入的 property。inject 函数接收两个参数，第一个参数是 property 的键 (String 类型)，第二个参数是 property 的默认值 (可选)。

<script>
import { inject } from 'vue'

export default {
  setup() {
    const userLocation = inject('location', 'The Universe')
    const userGeolocation = inject('geolocation')

    return {
      userLocation,
      userGeolocation
    }
  }
}
</script>

在 provide 定义依赖时，使用 ref 或 reactive 为组件添加响应式依赖。同时在 provide 中也可定义修改依赖的方法。另外为保证 provide 定义的依赖只能通过所定义的方法修改，可以使用 readonly 为组件提供只读的依赖。

<template>
  <MyMarker />
</template>

<script>
import { provide, reactive, readonly, ref } from 'vue'
import MyMarker from './MyMarker.vue

export default {
  components: {
    MyMarker
  },
  setup() {
    const location = ref('North Pole')
    const geolocation = reactive({
      longitude: 90,
      latitude: 135
    })

    const updateLocation = () => {
      location.value = 'South Pole'
    }

    provide('location', readonly(location))
    provide('geolocation', readonly(geolocation))
    provide('updateLocation', updateLocation)
  }
}
</script>

响应性 API

API 传入 返回 关键字
reactive plain-object 对象代理 对象
readonly plain-object or proxy 对象代理 只读
ref any { value: ... } 非对象
computed function { value: ... } 计算属性

reactive

通过 reactive 方法可以使对象具有响应式。reactive 函数接受一个对象，并返回一个响应式且可变的 proxy 对象代理。

注意:
reactive 函数返回的对象代理并不等于原始的对象。为保证对象的响应性，建议只使用对象的响应式代理，避免依赖原始对象。
同时不要使用 ES6 解构来获取响应式状态，否则状态将失去响应性。不过可以通过 toRefs 方法将响应式对象转换为一组 refs 再使用解构，同时这些 refs 将保留与源对象的响应式关联。

import { reactive, toRefs } from 'vue'

// 响应式状态
const book = reactive({
  author: 'Vue Team',
  year: '2020',
  title: 'Vue 3 Guide',
  description: 'You are reading this book right now ;)',
  price: 'free'
})

let { author, title } = toRefs(book)

title.value = 'Vue 3 Detailed Guide' // 需要使用 .value 作为标题，现在是 ref
console.log(book.title) // Vue 3 Detailed Guide

在渲染期间使用响应式状态，因为依赖跟踪的关系，当响应式状态改变时视图会自动更新。

对于嵌套的对象，reactive 函数可以深度代理对象中的所有成员。在返回的 proxy 中，所有嵌套 property 也都具有响应性。

isReactive

使用 isReactive 可以检查对象是否是由 reactive 创建的响应式 proxy。如果 proxy 是 readonly 创建的，但还包装了由 reactive 创建的另一个 proxy，它也会返回 true。

import { reactive, isReactive, readonly } from 'vue'
export default {
  setup() {
    const state = reactive({
      name: 'John'
    })
    const stateCopy = readonly(state)
    console.log(isReactive(stateCopy)) // -> true
  }
}

readonly

通过 readonly 方法可以使对象具有响应式，并且是只读的。readonly 函数接受一个响应式或纯对象，并返回一个响应式且只读的 proxy 对象代理。

import { reactive, readonly } from 'vue'

const original = reactive({ count: 0 })

// 只读的响应式状态
const copy = readonly(original)

console.log(original === copy) // false

watchEffect(() => {
  // 依赖 copy 的侦听器
  console.log(copy.count)
})

// 通过 original 修改 count，将会触发依赖 copy 的侦听器
original.count++

// 通过 copy 修改 count，将导致失败并出现警告
copy.count++ // 警告: Set operation on key 'count' failed: target is readonly.

使用只读的响应式状态，可以防止状态在应用程序的某个位置被更改，同时也可以跟踪响应式对象 (ref 或 reactive) 的变化。

对于嵌套的对象，readonly 函数可以深度代理对象中的所有成员。在返回的 proxy 中，所有嵌套 property 也都具有响应性，并且是只读的。

isReadonly

使用 isReadonly 可以检查对象是否是由 readonly 创建的只读 proxy。

isProxy

使用 isProxy 可以检查对象是否是由 reactive 或 readonly 创建的 proxy。

ref

通过 ref 方法可以使数据具有响应式。ref 函数接受一个任何类型的数据，并返回一个响应式且可变的 ref 对象引用。

如果将一个对象传入 ref 方法，则会通过 reactive 方法进行代理，使该对象具有高度的响应式。如果已经是代理，则会直接使用代理。

ref 对象具有指向内部值的单个 property .value，因此在组件其余部分中，需要加 .value property 来访问或更改 ref 内部值。而在组件模板中，通过代理可以直接获取 ref 内部值，而无需加 .value property。

<template>
  <h1>count:{{ count }}</h1>
  <p>
    <button @click="increase">increase</button>
  </p>
</template>

import { ref } from "vue";

function useCount() {
  let countRef = ref(0);

  console.log(countRef.value); // 0

  const increase = () => {
    countRef.value++;
  };
  return {
    count: countRef,
    increase
  };
}

export default {
  setup() {
    return {
      ...useCount();
    }
  }
}

注意: Vue3 的组件模板支持多个根节点。

当 ref 作为响应式对象的 property 被访问或更改时，为使其行为类似于普通 property，它会自动展开内部值。另外如果将新的 ref 赋值给现有 ref 的 property，将会替换旧的 ref。

const count = ref(0)
const state = reactive({
  count
})

console.log(state.count) // 0

state.count = 1
console.log(count.value) // 1

const otherCount = ref(2)

state.count = otherCount
console.log(state.count) // 2
console.log(count.value) // 1

Ref 展开仅发生在被响应式 Object 嵌套的时候。当从 Array 或原生集合类型如 Map 访问 ref 时，不会进行展开。

const books = reactive([ref('Vue 3 Guide')])
// 这里需要 .value
console.log(books[0].value)

const map = reactive(new Map([['count', ref(0)]]))
// 这里需要 .value
console.log(map.get('count').value)

实际上 ref 对象的 value 属性也指向一个 Proxy 对象，需要通过代理才能得到实例中的值。

ref 对象的特殊处理

isRef

使用 isRef 检查值是否为一个 ref 对象。

unref

unref 是 val = isRef(val) ? val.value : val 的语法糖。如果参数是一个 ref，返回内部值，否则返回参数本身。

toRef

toRef 可以用来为源响应式对象上的某个 property 新创建一个 ref。ref 可以被传递，它会保持对其源 property 的响应式连接。即使源 property 不存在，toRef 也会返回一个可用的 ref。这使得它在使用可选 prop 时特别有用，可选 prop 并不会被 toRefs 处理。

const state = reactive({
  foo: 1,
  bar: 2
})

const fooRef = toRef(state, 'foo')

fooRef.value++
console.log(state.foo) // 2

state.foo++
console.log(fooRef.value) // 3

toRefs

toRefs 将响应式对象转换为普通对象，其中结果对象的每个 property 都是指向原始对象相应 property 的 ref。toRefs 只会为源对象中包含的 property 生成 ref。如果要为特定的 property 创建 ref，则应当使用 toRef。

const state = reactive({
  foo: 1,
  bar: 2
})

const stateAsRefs = toRefs(state)

/*
stateAsRefs 的类型:
{
  foo: Ref<number>,
  bar: Ref<number>
}
*/

// ref 和原始 property 已经链接起来了
state.foo++
console.log(stateAsRefs.foo.value) // 2

stateAsRefs.foo.value++
console.log(state.foo) // 3

当从合成函数返回响应式对象时，均以 ref 的结果返回，就可以保证 setup 函数的返回结果中不包含 reactive 或 readonly 直接产生的数据，这样消费组件就可以在不丢失响应性的情况下对返回的对象进行解构或展开。

function useFeatureX() {
  const state = reactive({
    foo: 1,
    bar: 2
  })

  // 操作 state 的逻辑

  // 返回时转换为 ref
  return toRefs(state)
}

export default {
  setup() {
    // 可以在不失去响应性的情况下解构
    const { foo, bar } = useFeatureX()

    return {
      foo,
      bar
    }
  }
}

computed

通过 computed 方法可以使计算属性具有响应式。computed 函数接受一个 getter 函数作为第一个参数，并通过 getter 返回的值返回一个只读的响应式引用。

同时为了访问计算属性的值，需要像使用 ref 一样使用 .value property。访问时，会根据情况 (有缓存时不会再次运行) 决定是否要运行函数，但没有访问时，一定不会运行函数。

import { ref, computed } from 'vue'

const count = ref(1)
const plusOne = computed(() => count.value + 1)

console.log(plusOne.value) // 2

plusOne.value++ // error

同时 computed 函数也可以接受一个 setter 函数作为第二个参数，并通过 getter 以及 setter 返回的值返回一个可读写的响应式引用。

import { ref, computed } from 'vue'

const count = ref(1)
const plusOne = computed({
  get: () => count.value + 1,
  set: val => {
    count.value = val - 1
  }
})

plusOne.value = 1
console.log(count.value) // 0

watchEffect

为了根据响应式状态自动应用和重新应用副作用，可以使用 watchEffect 方法创建侦听器。它立即执行传入的副作用函数，同时响应式追踪其依赖，并在其依赖变更时重新运行该函数。

const count = ref(0)

watchEffect(() => console.log(count.value))
// -> logs 0

setTimeout(() => {
  count.value++
  // -> logs 1
}, 100)

当 watchEffect 在组件的 setup() 函数或生命周期钩子被调用时，侦听器会被链接到该组件的生命周期，并在组件卸载时自动停止。同时也可以显式调用返回值以停止侦听。

const stop = watchEffect(() => {
  /* ... */
})

// 停止侦听
stop()

当侦听器中传入了具有异步操作的副作用函数时，则需要在侦听器被停止或状态改变副作用即将重新执行时，清除失效的副作用。因此传入的副作用函数可以接收一个 onInvalidate 函数作为参数，它将用来注册清除失效副作用时的回调。并且清理函数必须要在异步操作之前被注册。

const data = ref(null)
watchEffect(async onInvalidate => {
  onInvalidate(() => {...}) // 在 Promise 解析之前注册清除函数
  data.value = await fetchData(props.id)
})

当侦听器依赖的状态多次改变时，为了避免副作用函数的重复调用，Vue 的响应性系统会缓存副作用函数，并异步地执行它们。另外在默认情况下，用户定义的副作用函数会在所有的组件 update 前执行。

通过传递带有 flush 选项的附加 options 对象，可以指定副作用函数的触发方式。flush 选项的默认值为 pre，表示副作用函数在组件 update 前执行，post 和 sync 分别表示副作用函数在组件 update 之后和同时执行。

watchEffect(
  () => {
    /* ... */
  },
  {
    flush: 'post'
  }
)

注意: 在组件更新后触发副作用，就可以访问更新的 DOM。但也将推迟副作用的初始运行，直到组件的首次渲染完成。

watch

watch 需要侦听特定的数据源，数据源可以是单个也可以是多个，并在单独的回调函数中执行副作用。默认情况下它是惰性的，回调仅在侦听源发生更改时调用。且当侦听器依赖的状态多次改变时，副作用函数只会调用一次。

// 侦听一个 getter
const state = reactive({ count: 0 })
watch(
  () => state.count,
  (count, prevCount) => {
    /* ... */
  },
  options
)

// 侦听一个 ref
const count = ref(0)
watch(count, (count, prevCount) => {
  /* ... */
}, options)

// 侦听多个数据
watch(
  [() => state.count, countRef],
  ([new1, new2], [old1, old2]) => {
    /* ... */
  },
  options
)

注意: 无论是 watchEffect 还是 watch，当依赖项变化时，回调函数的运行都是异步的 (微队列)。执行顺序: 同步 -> 异步 (微队列) -> 异步 (宏队列)

多数情况下建议选择 watchEffect 作为侦听器，而 watch 更适用于以下的场景:

  • 不希望回调函数一开始就执行。
  • 数据改变时，需要参考旧值。
  • 需要监控一些回调函数中不会用到的数据。

global state

由于 Vue3 的响应式系统本身可以脱离组件而存在，因此可以充分利用这一点，轻松制造多个全局响应式数据。但工程中如果有多个 Vue 应用时，所有的应用都将共享同一个数据。

global state 

// store/useLoginUser 提供共享数据

import { reactive, readonly } from "vue";

// 导入 API 模块
import * as userServ from "../api/user";

// 创建默认的全局单例响应式数据，仅供该模块内部使用
const state = reactive({ user: null, loading: false });

// 对外暴露的数据是只读的，不能直接修改
// 也可以使用 toRefs 进行封装，从而避免解构或展开后响应式丢失
export const loginUserStore = readonly(state);

// 对外暴露修改全局响应式数据的方法
export async function login(loginId, loginPwd) {
  state.loading = true;
  const user = await userServ.login(loginId, loginPwd);
  state.user = user;
  state.loading = false;
}

Provide & Inject

可以使用选项式 API 中的 provide 和 inject 配置，或组合式 API 中的 provide 和 inject 方法，在高层组件中提供共享数据，然后在后代组件中使用。

组件上的 Provide

如果要为不同的应用实例提供不同的共享数据，可以使用应用实例上的 provide 方法，设置一个可以被注入到应用范围内所有组件中的值，然后在应用实例上的组件中使用 inject 来接收 provide 的值。

creaetApp(App)
  .provide("foo", ref(1))
  .provide("bar", ref(2))
  .mount("#app");

应用上的 Provide 

// store/useLoginUser 创建一个共享数据

import { readonly, reactive, provide, inject } from "vue";

// 导入 API 模块
import * as userServ from "../api/user";

const key = Symbol();

// 向传入的 Vue 应用实例上提供共享数据
export function provideStore(app) {

  // 创建默认的响应式数据
  const state = reactive({ user: null, loading: false });

  // 创建修改响应式数据的方法
  async function login(loginId, loginPwd) {
    state.loading = true;
    const user = await userServ.login(loginId, loginPwd);
    state.user = user;
    state.loading = false;
  }

  // 对外暴露只读的数据和修改数据的方法
  app.provide(key, {
    state: readonly(state),
    login
  });
}

// 向组件上注入共享数据
export function useStore(defaultValue = null) {
  return inject(key, defaultValue);
}

// store/index

import { provideStore as provideLoginUserStore } from "./useLoginUser";
import { provideStore as provideNewsStore } from "./useNews"

export default function provideStore(app) {
  provideLoginUserStore(app);
  provideNewsStore(app);
}

// main.js

import { createApp } from "vue";
import App from "./App.vue"
import provideStore from "./store";
const app = createApp(App);
provideStore(app);
app.mount("#app");

响应性原理

Vue2 和 Vue3 均在相同的生命周期内完成数据响应式，Vue2 通过 Object.defineProperty 实现数据响应式，Vue3 则是通过 Proxy 来实现。

响应性原理

相较于 Object.defineProperty 的方式，通过 Proxy 实现数据响应式的效率更高。因为不必递归遍历所有属性，就可以直接得到一个 Proxy。所以在 Vue3 中，对数据的访问是动态的，当访问某个属性的时候，再动态的获取和设置，这极大的提升了在组件初始阶段的效率。

同时 Proxy 可以监控到成员的新增和删除，因此在 Vue3 中新增成员、删除成员、索引访问等均可以触发重新渲染，而这些在 Vue2 中是难以做到的。

效率提升

Vue3 的客户端渲染效率比 vue2 提升了 1.3~2 倍，SSR 渲染效率比 vue2 提升了 2~3 倍。

静态提升

Vue3 通过编译器将单文件组件的 template 模板编译成 render 函数，同时会将静态的节点 (没有绑定动态内容的元素节点) 以及属性提升到渲染函数外单独创建。

// vue2 的静态节点
render(){
  createVNode("h1", null, "Hello World")
  // ...
}

// vue3 的静态节点
const hoisted = createVNode("h1", null, "Hello World")
function render(){
  // 直接使用 hoisted 即可
}

每当响应式数据更改后，渲染函数都会再次运行，同时静态的节点以及属性也都会再次创建。但是静态内容并不会改变，多次创建会不但降低效率也会占用额外的空间。

<div class="user">
  {{user.name}}
</div>

const hoisted = { class: "user" }

function render(){
  createVNode("div", hoisted, user.name)
  // ...
}

静态提升后，所有的静态节点以及属性只会被创建一次，并可以在渲染函数中重复使用。

预字符串化

对于大多是静态内容的组件，Vue3 通过编译器将大量连续的静态内容编译成一个普通字符串节点。

<div class="menu-bar-container">
  <div class="logo">
    <h1>logo</h1>
  </div>
  <ul class="nav">
    <li><a href="">menu</a></li>
    <li><a href="">menu</a></li>
    <li><a href="">menu</a></li>
    <li><a href="">menu</a></li>
    <li><a href="">menu</a></li>
  </ul>
  <div class="user">
    <span>{{ user.name }}</span>
  </div>
</div>

const _hoisted_2 = _createStaticVNode("<div class=\"logo\"><h1>logo</h1></div><ul class=\"nav\"><li><a href=\"\">menu</a></li><li><a href=\"\">menu</a></li><li><a href=\"\">menu</a></li><li><a href=\"\">menu</a></li><li><a href=\"\">menu</a></li></ul>")

预字符串化可以将静态节点与动态节点分离。在重新渲染时，只有动态节点会被重新创建，从而有效降低虚拟节点的创建数量。特别是在 SSR 中效果提升非常明显。

缓存事件处理函数

Vue3 通过编译器将单纯的事件处理函数缓存起来，以便下次使用。这些事件函数不会改变，因而只需生成一次即可。

<button @click="count++">plus</button>

// vue2
render(ctx){
  return createVNode("button", {
    onClick: function($event){
      ctx.count++;
    }
  })
}

// vue3
render(ctx, _cache){
  return createVNode("button", {
    onClick: cache[0] || (cache[0] = ($event) => (ctx.count++))
  })
}

Block Tree

Vue3 通过编译器在 Block 节点 (根节点) 的数组中记录所有后代的动态节点。当重新渲染对比时无需把所有的节点进行对比，只要通过 PatchFlag 找到 Block 节点中记录的动态节点依次对比即可。

<form>
  <div>
    <label>账号：</label>
    <input v-model="user.loginId" />
