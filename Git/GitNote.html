Git 笔记

git 配置

查看配置

  • 查看配置信息
    git config --list
  • 查看特定配置信息
    git config [environment variable]

用户信息配置

  • 配置用户名
    git config --global user.name "[name]"
  • 配置用户邮箱
    git config --global user.email "[email]"

其他配置

  • 启用命令行输出的彩色化
    git config --global color.ui true
  • 配置git命令别名
    git config --global alias.[alias] [command]

--system: 读写/etc/gitconfig 文件：系统中对所有用户都普遍适用的配置。
--global: 读写~/.gitconfig 文件：用户目录下的配置文件只适用于该用户。

如果要在某个特定的项目中使用其他名字或者电邮，只要去掉 --global 选项重新配置即可，新的设定保存在当前项目的 .git/config 文件里。

初始化仓库

使用指定目录作为 Git 仓库

git init [project name]
cd [project name]

使用当前目录作为 Git 仓库

mkdir [project name]
cd [project name]
git init

git 工作流

本地仓库由 git 维护的三棵“树”组成。第一个是 工作区(WorkingDir)，它持有实际文件，也就是电脑里的目录；第二个是 暂存区(Index)，它像个缓存区域临时保存改动，一般存放在 ".git 目录下" 下的 index 文件中；最后是 游标所指向的分支(HEAD)，它指向最后一次提交的结果。

sequenceDiagram participant A as 工作区 participant B as 暂存区 participant C as HEAD分支 par commit -a A->>B: add <file> B->>C: commit end par checkout HEAD <file> / reset --hard (<commit>) C->>B: reset (HEAD) (<file>) B->>A: checkout -- <file> end par rm <file> A->>A: 删除文件 B->>B: rm --cached <file> end

将工作区文件添加到暂存区

git add [file name / *]

-f: 强制添加到Git，例如添加到.gitignore文件中的忽略文件

将暂存区改动提交到HEAD分支

git commit -m "[descriptive message]"

将工作区改动添加到暂存区并提交到HEAD分支

git commit -am "[descriptive message]"

将暂存区(文件)回退到HEAD分支中最近的版本

git reset (HEAD) ([file name])

将工作区和暂存区切换到HEAD分支中(指定)的版本

git reset --hard ([commit id])

[commit id]: 当前版本 HEAD / 上一个版本 HEAD^ / 上上一个版本 HEAD^^ / 往上100个版本 HEAD~100 / 版本号

将工作区文件回退到暂存区中最近的版本

git checkout -- [file name]

将工作区和暂存区的文件切换到HEAD分支中最近的版本

checkout HEAD [file name]

将文件从暂存区删除

git rm --cached [file name]

将已提交HEAD分支且未修改的文件从工作区和暂存区删除

git rm [file name]

-f: 不论文件是否修改过或提交过HEAD分支，强制删除工作区和暂存区的文件
-r: 递归删除目录文件

移动或重命名文件、目录、软连接

git mv [source directory / original name] [destination directory / modified name]

git 查看命令

查看工作区当前的状态

git status

-s: 以简短格式输出

查看已提交的文件或分支工作区与暂存区的差异

git diff ([brach name / file name])

--stat: 只显示摘要
--cached / --staged: 查看暂存区与HEAD分支的差异

查看已提交的文件或分支工作区与HEAD分支的差异

git diff HEAD ([brach name / file name])

查看提交历史记录

git log

--oneline: 简洁显示
--graph: 图形显示
--reverse: 倒叙显示
--since=[date] / --after=[date]: 显示特定日期之后的提交
--until=[date] / --before=[date]: 显示特定日期之前的提交
--author=[name] / --committer=[name]: 显示特定用户的提交
--no-merges: 不显示分支的提交
--follow [file name]: 显示文件的版本历史记录，包括重命名

查看操作历史记录

git reflog

显示指定提交的元数据和内容更改

git show [commit id / tag name]

显示此项目中所有被忽略的文件(.gitignore中的文件)

git ls-files --other --ignored --exclude-standard

git 标签

查看标签

  • 查看所有标签
    git tag
  • 查看标签详细信息
    git show [tag name]

设置标签

git tag [tag name] ([commit id])

-a: 创建添加无符号、带注释的标签
-s: 创建添加 PGP 签名的标签
-d: 删除标签

github 远程仓库

配置SSH key

  1. 创建SSH Key
    ssh-keygen -t rsa -C "[email address]"
  2. 进入 ~/ 下生成 .ssh 文件夹，打开 id_rsa.pub，复制里面的 key
  3. 登陆 GitHub，打开“Account settings”，“SSH Keys”页面 Add Key

通过ssh -T git@github.com验证是否成功

查看远程仓库

git remote

-v: 显示详细信息

添加远程仓库

git remote add [repo alias] [url]

例如: git remote add origin git@github.com:test/test.git

删除远程仓库

git remote rm [repo alias]

本地库推送到远端库

git push [repo alias] ([branch name / tag name])

例: 把本地分支master推送到远端库的master分支 git push origin master

-u: 在把本地分支内容推送的远端库的master分支的同时，把本地的分支和远端的master分支关联起来
--tags: 推送全部尚未推送到远端库的本地标签
:refs/tags/[tag name]: 删除远端的标签

远端库拷贝到本地库

git clone [repo url] ([directory])

例: 拷贝 github 中 jquery 代码仓库到当前目录下的 myJquery 目录

  • 使用 HTTPS 协议拷贝
    git clone https://github.com/jquery/jquery.git myJquery
  • 使用 SSH 协议拷贝
    git clone git@github.com:jquery/jquery.git myJquery

注: 从远端仓库拷贝时，实际上Git自动把本地的master分支和远端的master分支对应起来，并且，远端仓库的默认名称是origin。

提取远端库的更新

  • 从远端库下载对象和引用
    git fetch [repo alias]
    将更新同步到本地
    git merge [repo alias]/[branch]
  • 从远端库下载对象和引用并将更新同步到本地
    git pull ([repo alias] [branch])

--allow-unrelated-histories: 合并两个独立仓库的历史 (当没有使用git clone将远端库拷贝到本地时，使用git pull会拒绝合并不相关的历史，原因是本地库和远端库是独立的两个仓库没有关联)

git 分支

创建分支

git branch [branch name]

切换分支

  • git checkout [branch name]
  • git switch [branch name]

创建并切换分支

  • git checkout -b [branch name] ([start point])
  • git switch -c [branch name] ([start point])

注: start point 以某个分支为起点创建分支，默认以当前分支为起点，可以设置远程库分支为起点[repo alias]/[branch]

查看分支

git branch

--set-upstream-to=[remote] [branch]: 指定本地分支与远程分支[repo alias]/[branch]的链接

合并分支 (Fast forward 模式)

git merge [branch name]

注: 删除分支后，会丢掉分支信息

合并分支 (recursive 模式)

git merge --no-ff -m "[descriptive message]" [branch name]

注: 生成一个新的 commit，从分支历史上可以看出合并的分支信息，通过 git log --graph 查看

删除分支

git branch -d [branch name]

-D: 对于未合并过的分支执行强制删除

git 片段

临时存储工作区和暂存区所有交由git管理的已修改的片段

git stash

注: git未跟踪的文件将继续保留在工作区中

列出所有临时存储的片段

git stash list

恢复最近或指定的临时存储片段

git stash apply ([stash id])

删除最近或指定的临时存储片段

git stash drop ([stash id])

恢复并删除最近或指定的临时存储片段

git stash pop ([stash id])

git 其他操作

应用已提交的更改 (bug修复)

git cherry-pick [commit id]

把本地未push的分叉提交历史整理成直线

git rebase

注: 通过 git log --graph 查看

调试 .gitignore 中排除的文件

git check-ignore

-v: 将排除模式与路径一起打印
